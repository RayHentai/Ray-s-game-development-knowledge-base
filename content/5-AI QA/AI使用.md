# 🚀 AI 赋能程序员 · 完整学习路径

## 第一阶段：掌握 AI 对话的核心技巧

### 1. 理解 AI 的"思维方式"

AI 本质上是一个**根据你的输入预测最佳输出**的系统。你给的信息越精准，它的输出越好。

新手最常犯的错误：

```
❌ 糟糕的提问："帮我写个登录功能"

✅ 好的提问："用 Python + FastAPI 写一个 JWT 登录接口，
   需要：邮箱+密码验证、返回 access_token、
   错误时返回标准 JSON 格式，代码要有注释"
```

---

### 2. 提示词的黄金公式

```
角色 + 背景 + 任务 + 格式 + 限制
```

**角色（Role）**：告诉 AI 它是谁

> "你是一个有 10 年经验的 Python 后端工程师"

**背景（Context）**：告诉 AI 你的情况

> "我是初学者，项目用的是 FastAPI + PostgreSQL"

**任务（Task）**：清晰描述你要什么

> "帮我写一个用户注册接口"

**格式（Format）**：规定输出形式

> "代码 + 注释 + 使用示例"

**限制（Constraint）**：边界条件

> "不要用第三方库，代码不超过 50 行"

---

### 3. 进阶技巧：让 AI 输出更精准

**技巧一：给例子**

```
输入示例：{"email": "test@test.com"}
输出示例：{"token": "xxx", "expires_in": 3600}
```

**技巧二：让 AI 先思考再回答**

> "先分析这个问题的难点，再给出解决方案"

**技巧三：分步拆解复杂任务**

> 不要一次问完所有东西，先问架构，再问实现，再问优化

**技巧四：反馈迭代**

> "你上面的代码有个问题：XXX，请修改第 20 行的逻辑"

**技巧五：让 AI 扮演你的用户/审查者**

> "现在假设你是一个挑剔的代码审查员，找出上面代码的问题"

---

### 4. 不同场景的提示词模板

| 场景    | 模板示例                              |
| ----- | --------------------------------- |
| 学习概念  | "用简单的比喻解释 XXX，然后给一个实际代码例子"        |
| Debug | "这段代码报错：[错误信息]，代码是：[代码]，帮我找原因"    |
| 代码审查  | "审查这段代码的：性能、安全性、可读性，给出改进建议"       |
| 方案选型  | "我要做 XXX，对比 A 方案和 B 方案的优缺点，我是初学者" |

## 第二阶段：AI 赋能写代码（进阶）

等你熟悉了上面的技巧，我们再深入这些方向：

- **用 AI 辅助学习**：让 AI 做你的私教，针对性讲解
- **用 Claude Code**：在终端直接用 AI 写/改代码
- **需求 → 代码的完整流程**：从模糊需求到可运行项目
- **AI 做代码审查 & 重构**：提升代码质量
- **自动化测试用例生成**：AI 帮你写测试
