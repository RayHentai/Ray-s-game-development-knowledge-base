## 一、游戏概览

### 电梯简介
> 你是一名星际工程师，独自降落在一颗颗未知星球上。采集资源、建立自动化生产线、在怪物的围攻中研发科技、制造火箭飞向下一颗星球——每颗星球都是一段新的征途。

### 核心三支柱
| 支柱 | 体验目标 | 参考 |
|------|---------|------|
| **探索** | 每颗星球有独特资源和任务链，驱动玩家向前 | Minecraft整合包任务系统 |
| **建造** | 自动化生产线 + 能源管理带来"工厂运转"的成就感 | Factorio、Shapez |
| **生存** | 战斗带来紧张感，污染制造长期压力 | Vampire Survivors |

## 二、核心游戏循环

```
降落星球
    ↓
点击采集点获取自然资源（进背包）
    ↓
建造机器（从背包消耗材料）
    ↙                         ↘
加工机器（矿石→中间产物→锭）    采集机器（自动采矿/地形抽取）
    ↓                              ↓
输出缓存 → 玩家取走 → 背包      输出缓存 → 玩家取走 → 背包
    ↘                         ↙
       消耗资源 + 能源管理
       （电力/燃料/污染三角平衡）
              ↓
         升级科技树
              ↓
     挑战地牢Boss → 获得火箭设计图
              ↓
         制造火箭·解锁新星球
              ↓
           （循环）
```

## 三、资源系统

> 详见 [[资源系统-地球篇]]

### 框架分层
| 框架 | 内容 |
|------|------|
| **自然资源** | 星球表面存在的原始材料，玩家点击采集点获取 |
| **加工产物** | 机器加工出的中间产物/成品，进机器输出缓存 |

### 资源分类（ResourceTier）
| 枚举值 | 含义 | 示例 |
|--------|------|------|
| `Ore` | 单质矿石，矿藏有限 | 铁矿石、铜矿石 |
| `Natural` | 可采集自然资源 | 木头 |
| `Terrain` | 地形采集，靠近特定地形 | 水、原油、熔岩 |
| `Gem` | 宝石类，特殊加工路线 | 钻石矿石 |
| `Simple` | 无需处理，直接使用 | 沙子、石灰石、石材 |
| `Special` | 战斗/任务掉落 | 星核碎片 |
| `Crushed`~`Ingot` | 各层加工产物 | 粉碎矿石→矿粉→金属锭 |

### 地球自然资源（19种）
**单质矿石（普通）：** 煤炭、铜、锡、铁  
**单质矿石（稀有）：** 金、硅、铝土、硫磺  
**地形采集：** 熔岩（矿藏有限）、水（无限）、原油（有限）、天然气（有限）  
**可采集：** 木头（可再生，砍树有概率掉种子）  
**无需处理：** 石材、沙子、石灰石  
**宝石类：** 钻石矿石（深层+地牢+精英怪）

### 命名规范（物流过滤器依赖）
```
ore_[矿]           原矿        ore_iron
crushed_[矿]       粉碎矿石    crushed_iron
ore_washed_[矿]    洗净矿石    ore_washed_iron
powder_impure_[矿] 含杂粉      powder_impure_iron
powder_washed_[矿] 矿粉        powder_washed_iron
powder_clean_[矿]  洁净粉      powder_clean_iron
ingot_[矿]         金属锭      ingot_iron
crystal_[矿]       晶体        crystal_silicon
gem_[宝石]         宝石成品    gem_diamond
```

|------|---------|
| 木头 | 30 |
| 煤炭矿石 | 80 |
| 原油 | 200 |

## 四、加工系统

> 详见 [[自动化系统]]（待补充加工产物完整链条）

### 加工建筑类型
| 建筑 | 类型枚举 | 用途 |
|------|---------|------|
| 粉碎机 | `Crusher` | 矿石→粉碎矿石/含杂粉 |
| 洗矿机 | `Washer` | 粉碎矿石→洗净矿石/矿粉 |
| 熔炉 | `Smelter` | 矿粉/洁净粉→金属锭（铝土/硅不可用）|
| 高炉 | `BlastFurnace` | 铝土/硅专用，需科技树解锁 |
| 筛选机 | `Sorter` | 宝石矿石→宝石成品 |

### 矿石加工路线（正常矿石）
```
路线①  矿石 → 熔炉 → 金属锭×1（无增产）
路线②  矿石 → 粉碎 → 含杂粉 → 熔炉 → 金属锭×1
路线③  矿石 → 粉碎 → 含杂粉 → 洗矿 → 矿粉 → 熔炉 → 金属锭×1
路线④  矿石 → 粉碎 → 洗矿 → 洁净粉 → 熔炉 → 金属锭×2（最高效）
```

### 副产物规则
| 加工步骤 | 主产物副产物 |
|---------|------------|
| 粉碎机→粉碎矿石 | 副产物：石材（100%）|
| 粉碎机→含杂粉/洁净粉 | 副产物：自身矿粉（10%）|
| 洗矿机→洗净矿石 | 副产物：石材（100%）|
| 洗矿机→矿粉 | 副产物：自身矿粉（10%）|
| 筛选机 | 副产物：宝石×1（10%）|

> 副产物进机器输出缓存独立槽位，不直接进背包

### 机器缓存系统
```
输入缓存  玩家手动投料（从背包扣除）→ 满则不接受
加工中    消耗输入缓存，计时
输出缓存  主产物+副产物各占独立槽位，总量满则停机
玩家取货  从输出槽取走 → 进背包（InventoryManager）
```

## 五、能源系统

### 电力系统（MVP：无线电网）
- 全星球共享一张无线电网
- 用电超过发电85%→警告；超过100%→短路，用电建筑停机
- 单位：A（安培，游戏自定义）

### 燃料系统
- 过渡采矿机（燃料驱动）→ 直接产出成品，无需加工
- 电力采矿机 → 产出矿石，需走加工路线
- 热值（HV）/ 用电当量（A）= 运行秒数

### 污染系统
- 全局污染值 0–100，燃料建筑产生较高污染
- 污染≥50：怪物强度+20%
- 污染≥80：采集效率-30%

### v2.0预留
- 有线电网 + 电线杆（初/高级）
- 无线收发芯片系统
- 多种发电机（火力·风力·太阳能·热能·核能）
- 污染净化建筑

## 六、战斗系统

> 类吸血鬼幸存者，v2.0替换为俯视角主动攻击

- WASD移动，武器自动攻击最近敌人
- 每击杀N只→弹出3选1被动词条升级
- 波次机制：每60秒强度提升
- 污染值影响怪物强度

**MVP词条池（6个）：**
快速射击 · 穿透弹 · 范围爆炸 · 磁力场 · 护盾再生 · 超载模式

**掉落：** 普通怪→矿石类；精英怪（每5波）→星核碎片+低概率钻石

## 十、技术架构

### Manager 全景
```
GameManager（场景入口）
├── InventoryManager   玩家背包（所有进出背包都经过这里）✅
├── ResourceManager    世界数据（矿藏储量/全局追踪）✅
├── BuildingManager    建筑建造·费用·计数 ✅
├── PowerManager       电力发电·用电·短路 ✅
├── FuelManager        燃料热值·燃烧消耗 ✅
├── PollutionManager   全局污染值·阈值惩罚 ✅
├── EnemySpawner       波次控制·怪物生成（待实现）
├── CombatManager      战斗·词条系统（待实现）
├── TechTreeManager    科技树状态·解锁（待实现）
├── QuestManager       任务追踪·奖励（待实现）
├── PlanetManager      星球切换·存档（待实现）
└── UIManager          所有UI面板控制（待实现）
```

### 资源流向
```
采集点（ClickCollector）
  玩家点击     → 消耗矿藏 → InventoryManager
  机器自动采集  → 进采集点缓存 → 机器取走 → InventoryManager

地形采集机（TerrainExtractor）
  连接采集点    → 自动抽取 → 输出缓存
  玩家取货      → InventoryManager

加工机（ProcessingBuilding）
  玩家投料      → InventoryManager扣除 → 输入缓存
  加工完成      → 输出缓存（主产物+副产物独立槽位）
  玩家取货      → InventoryManager
  副产物满溢出  → 丢弃

建造费用       → InventoryManager扣除
```

### 架构原则
- **单一数据源**：每类数据只有一个Manager管理
- **事件驱动**：数据变化广播事件，不轮询
- **数据驱动**：ScriptableObject存配置，Manager处理逻辑
- **状态机**：机器运行状态用State Machine管理
- **策略模式**：RecipeData是策略，切换配方不改机器代码
