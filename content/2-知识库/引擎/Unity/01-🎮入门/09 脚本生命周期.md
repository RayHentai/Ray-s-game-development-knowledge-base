## 帧的概念

**帧：每秒钟帧数**
- 1s = 1000ms
- 60帧 ≈ 16.66ms
- 30帧 ≈ 33.33ms

## 核心理解

> 就是该脚本对象依附的 GameObject 对象从出生到消亡整个生命周期中，会通过反射**自动调用**的一系列函数。
> Unity 帮助我们记录了一个 GameObject 对象依附了哪些脚本，在游戏执行的**特定时机**，通过函数名反射得到并执行一些固定名字的函数。

---

## 生命周期函数执行顺序

**访问修饰符：** 
- private、protected 
- 如果不写默认为private
- 一般不写public 因为不需要在外部自己调用生命周期函数 都是Unity自己帮助我们调用的

```csharp
Awake        → 类对象自己被创建时调用（脚本实例化时）
    ↓
OnEnable     → 激活时调用（可多次调用）
    ↓
Start        → 第一次帧更新之前调用（只调用一次）
    ↓
    ┌───────────────────────────────────────────┐
    │ FixedUpdate  → 独立于游戏帧，固定时间调用   │  ← 循环
    │              （用于物理更新）              │
    │              时间间隔可在 Project Settings │
    │              → Time 中设置                 │
    └───────────────────────────────────────────┘
    ↓
    ┌───────────────────────────────┐
    │ Update      → 每帧调用         │  ← 循环
    │ LateUpdate  → 晚于 Update 每帧 │
    └───────────────────────────────┘
    ↓
OnDisable    → 失活时调用（可多次调用）
    ↓
OnDestroy    → 销毁类对象时调用
```

---

## 各函数用途

>生命周期函数支持继承多态

如果不打算在生命周期中写逻辑，那就不要声明这些生命周期函数 
因为通过反射默认会调用，调用空函数多此一举

| 函数            | 时机              | 用途                                                       |
| ------------- | --------------- | -------------------------------------------------------- |
| `Awake`       | 当（这个类对象自己）被创建时  | 初始化（比 Start 早，不依赖其他脚本），只执行一次                             |
| `OnEnable`    | GameObject每次激活时 | 激活时的逻辑                                                   |
| `Start`       | 第一次帧更新之前        | 初始化（可以依赖其他 Awake 已执行的逻辑），只执行一次                           |
| `FixedUpdate` | 每一物理帧都执行        | 物理相关更新（固定帧率，默认0.02s/次）时间间隔 可以在 Project Setting中的 Time去设置 |
| `Update`      | 每帧执行            | 每帧逻辑（大多数游戏逻辑写这里）                                         |
| `LateUpdate`  | 在UpDate之后 每帧执行  | 每帧逻辑，在 Update 之后执行（适合摄像机跟随）                              |
| `OnDisable`   | 失活时             | 失活时的逻辑                                                   |
| `OnDestroy`   | 删除对象时           | 销毁时的清理逻辑                                                 |

---

## 注意事项

- **每次激活时** 会调用 OnEnable（不是只调用一次）
- `FixedUpdate` 每帧调用的次数和游戏帧率无关，取决于物理时间步长
- `Update` 每帧调用，帧率不稳定所以时间间隔不固定
- `LateUpdate` 适合做摄像机跟随（保证在角色移动后再更新摄像机）
- 对象在被删除时会先失活，可以在 `OnDisable` 中处理一些删除对象的逻辑
- **运行时失活状态下的对象不会执行**任何生命周期函数
- `GameObject.activeSelf` 为 false 时，不执行

## 练习

> 1. 请默写出各个生命周期函数并写出他们分别何时执行以及执行的先后顺序

```csharp
① Awake       — 类对象自己被创建时调用                 ↓
② OnEnable    — 激活时调用（可多次调用）               ↓
③ Start       — 第一次帧更新之前调用                   ↓
④ FixedUpdate — 独立于游戏帧，每帧调用（物理更新用 ← ↑  ↓
⑤ Update      — 每帧调用                           ↑  ↓
⑥ LateUpdate  — 晚于 Update 每帧调用             → ↑  ↓
⑦ OnDisable   — 失活时调用（可多次调用）               ↓
⑧ OnDestroy   — 销毁类对象时调用                      ↓
```

> 2. 生命周期函数并不是基类中的成员，为什么 Unity 可以自动执行这些特殊的函数？


- 反射。Unity 记录了所有场景的 Object 对象和关联的脚本对象。
- 在游戏执行的特定时机，Unity 会通过函数名反射得到脚本对象中生命周期对应的函数，然后在特定时期执行它们。
