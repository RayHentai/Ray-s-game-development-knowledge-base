# ⚖️ 四种持久化方案横向对比

> 一页搞清楚四种方案的区别，选型直接查这里。

---

## 一句话定位

| 方案 | 一句话 |
|------|--------|
| **PlayerPrefs** | 存少量简单配置，用键值对，三行代码搞定 |
| **XML** | 有层级的配置数据，人可以直接看懂和编辑 |
| **Json** | 主流的数据交换格式，和服务器通信首选 |
| **二进制** | 性能最好、体积最小、最难被破解，用于存档/大量数据 |

---

## 核心对比表

| 对比项 | PlayerPrefs | XML | Json | 二进制 |
|--------|------------|-----|------|--------|
| **数据类型支持** | ❌ 只有 int/float/string | ✅ 任意（需映射） | ✅ 任意 | ✅ 任意 |
| **存储复杂对象** | 需手动拆分字段 | ✅ 序列化支持 | ✅ 序列化支持 | ✅ 序列化支持 |
| **存储 List** | 需手动存数量+循环 | ✅ | ✅ | ✅ |
| **存储 Dictionary** | 需手动处理 | 麻烦 | ✅（LitJson）| ✅ |
| **人类可读** | ⚠️ 注册表/plist | ✅ 直接看懂 | ✅ 直接看懂 | ❌ 乱码 |
| **文件大小** | 极小 | 大（标签冗余）| 中等 | 最小 |
| **读写性能** | 最快（内存）| 慢 | 中等 | 最快 |
| **安全性** | ❌ 注册表可直接改 | ❌ 文本可直接改 | ❌ 文本可直接改 | ✅ 加密后难破解 |
| **跨平台** | ✅（Unity处理）| ✅ | ✅ | ✅ |
| **与服务器通信** | ❌ | ⚠️ 少用 | ✅ 主流格式 | ⚠️ 需约定协议 |
| **代码复杂度** | ⭐ 最简单 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |

## 存储位置

| 方案 | Windows | Android | iOS |
|------|---------|---------|-----|
| PlayerPrefs | 注册表 `HKCU\Software\公司\产品` | `/data/data/包名/shared_prefs/` | `/Library/Preferences/应用ID.plist` |
| XML / Json / 二进制 | `Application.persistentDataPath` 或自定义路径 | 同左 | 同左 |

---

## 适用场景速查

**用 PlayerPrefs 当：**
- 存储少量简单设置（音量、画质、语言）
- 存储单个数值（最高分、金币数）
- 原型阶段快速验证，不想写文件 I/O

**用 XML 当：**
- 数据需要策划/非程序员直接编辑
- 需要有层级结构的配置文件（技能树、关卡配置）
- 项目已有 XML 数据源需要读取

**用 Json 当：**
- 和服务器通信（REST API 标准格式）
- 数据结构复杂，有嵌套对象和数组
- 需要 Dictionary 存储
- 推荐作为**默认本地存储方案**

**用二进制 当：**
- 对安全性有要求（防止玩家修改存档作弊）
- 数据量大，需要最小文件体积和最快读写
- 需要自定义加密（配合异或/AES等算法）

## 序列化方案对比

| | JsonUtility | LitJson | XmlSerializer | BinaryFormatter |
|-|------------|---------|--------------|----------------|
| **所属** | Unity 内置 | 第三方库 | .Net 内置 | .Net 内置 |
| **Dictionary 支持** | ❌ | ✅ | ❌ | ✅ |
| **需要 [Serializable]** | ✅ 必须 | ❌ | ✅ 必须 | ✅ 必须 |
| **支持私有字段** | ✅（加 [SerializeField]）| ❌ | ❌ | ✅ |
| **性能** | 快 | 中等 | 慢 | 快 |
| **输出格式** | Json 文本 | Json 文本 | XML 文本 | 二进制 |
| **适用场景** | Unity 项目本地存储 | 需要 Dictionary / 复杂结构 | 配置文件 | 安全性要求高的存档 |

## 选型决策树

```
需要和服务器通信？
  └── 是 → Json（LitJson）

数据有 Dictionary？
  └── 是 → Json（LitJson）或 二进制

数据需要策划直接编辑？
  └── 是 → XML 或 Json

对安全性有要求？
  └── 是 → 二进制 + 加密

只是几个简单设置/数值？
  └── 是 → PlayerPrefs

以上都不是，需要存复杂对象？
  └── 默认选 Json（JsonUtility 或 LitJson）
```
