# 🔐 04 加密思路

**所属模块**：[[00 PlayerPrefs 总览]]
**查阅次数**：0

---

## 核心思路

> PlayerPrefs 数据存在本地，容易被玩家找到并修改（作弊）。
> 通过加密手段，让数据"找不到"、"看不懂"、"解不出"。

---

## 三个层次

### 找不到（隐藏位置）

- **多层文件包裹**：把数据藏在不明显的目录层级中
- **利用服务器存数据**：关键数据存服务器，本地只存临时缓存

### 看不懂（混淆数据）

- **对数据进行特殊处理**：存储时转换，读取时还原
- **名字辨识度低**：key 名使用无意义的字符串（如哈希值）

### 解不出（加密算法）

- **复杂的解密规则**：使用 MD5、AES、Base64 等加密算法

---

## 简单加密实践

```csharp
// 思路：存储时加密，读取时解密

// 存储时：对值进行偏移处理（简单示例）
int realAtk = 999;
PlayerPrefs.SetInt("atk", realAtk + 1000); // 存 1999

// 读取时：还原真实值
int atk = PlayerPrefs.GetInt("atk") - 1000; // 取出 1999 - 1000 = 999

// 或者使用 XOR 异或加密（更安全，同一操作可以加密也可以解密）
int key = 12345;
int realAtk = 999;
int encoded = realAtk ^ key; // 加密存储
int decoded = encoded ^ key; // 解密读取（decoded == 999）
PlayerPrefs.SetInt("atk", encoded);
int atk = PlayerPrefs.GetInt("atk") ^ key;
```
