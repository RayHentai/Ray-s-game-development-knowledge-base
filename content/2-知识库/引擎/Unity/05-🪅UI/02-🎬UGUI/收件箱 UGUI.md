## 整理规则

遇到以下情况，按对应方式处理：

| 内容类型 | 移到哪里 |
|----------|----------|
| 某个知识点没搞懂 | → 对应的 `.md` 文件的「踩坑记录」 |
| 一个新的语法/用法 | → 对应知识点的「注意事项」或单独新建文件 |
| 做练习时的思路 | → 对应知识点的「练习」区域 |
| 老师说的一句话/技巧 | → 编程思维 或 补充知识点 |
| 查阅次数要更新 | → 本阶段总览的统计表 |
| 和本阶段无关的内容 | → 移到其他阶段的收件箱，或直接删除 |
