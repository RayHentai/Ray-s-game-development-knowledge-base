## 核心理解

> 简称协程，它是“假”的多线程，不是多线程

作用：
- 将代码简单化，不卡主线程
- 把可能会让主线程卡顿的耗时的逻辑分时分布执行

---

## 协程 和 线程的区别

| 对比项  | 协程（Coroutine）       | 线程（Thread）      |
| ---- | ------------------- | --------------- |
| 执行方式 | 在原线程之上开启，进行逻辑分时分步执行 | 独立于主线程，和主线程并行执行 |

## 语法 / 用法

> 继承MonoBehavior的类都可以开启协程函数

### 协程的声明与开启

**注意：**
- 协程开启后
- 组件和物体销毁，协程不执行
- 物体失活协程不执行，**组件失活协程执行**

```csharp
//1 声明协程函数
// 1.1 返回值必须为IEnumerator类型及其子类
// 1.2 函数中通过tield return 返回值; 进行返回
IEnumerator MyCoroutine(int i, string str)
{
	print(i);
	yield return new WaitForSeconds(5f);//等待5秒
	print(str);
	yield return new WaitForSeconds(1f);//写几个tield return 就代表把代码分成了几部分
	print("142");
	
	while (true)//协程也可以写成死循环 把代码分成无限步
	{
		Print("345");
		yield return new WaitForSeconds(1f);
	}
}

//2 开启协程函数
MyCoroutine(1, "123");//协程函数 是不能这样执行的 不会有任何效果
//常用开启方式
IEnumerator ie = MyCoroutine(1, "123");
StartCoroutine(ie);

Coroutine c1 = StartCoroutine(MyCoroutine(1,"123"));//和上面是一样的 一步到位写法
Coroutine c2 = StartCoroutine(MyCoroutine(1,"123"));
Coroutine c3 = StartCoroutine(MyCoroutine(1,"123"));//协程可以开启多个

//3 关闭协程函数
//关闭所有协程
StopAllCoroutines();

//关闭指定协程
StopCoroutines(c1);
```

### yield return 不同内容的含义

```csharp
//1.下一帧执行
yield return 数字;
yield return null;
//在Update和LateUpdate之间执行

//2.等待指定秒后执行
yield return new WaitForSeconds(秒);
//在Update和LateUpdate之间执行

//3.等待下一个固定物理帧更新时执行
yield return new WaitForFixedUpdate();
//在FixedUpdate和碰撞检测相关函数之后执行

//4.等待摄像机和GUI渲染完成后执行
//主要在截图时使用
yield return new WaitForEndofFrame();
//在LateUpdate之后的渲染相关处理完毕后之后

//5.一些特殊类型的对象比如异步加载相关函数返回的对象
//之后讲解异步加载资源异步加载场景 网络加载时再讲解
//一般在Update和LateUpdate之间执行

//6.跳出协程
yield break;
```

## 协同程序的原理

> 协程的本体本质上就是一个C#的迭代器方法
> C#的迭代器函数"分步执行"的特点，加上协程调度逻辑，实现的一套分时执行函数的规则

**协程可以分成两部分**
1. 协程函数本体 IEnumerator Name() { yield return null }
2. 协程调度器 StartCoroutine

协程本体就是一个能够中间暂停返回的函数

协程调度器是unity内部实现的，会在对应的时机帮助我们继续执行协程函数
unity只实现了协程调度部分

```csharp
public TestClass
{
	public int time;
	public TestClass(int time) { this.time = time; }
}

//1 协程函数本体
IEnumerator Test()
{
	Print("第一次执行")
	yield return 1;
	Print("第二次执行")
	yield return 2;
	Print("第三次执行")
	yield return "123";
	Print("第四次执行")
	yield return new TestClass(10);
}
//如果我们不通过开启协程方法执行协程
//Unity的协程调度器是不会帮助我们管理协程函数的
IEnumerator ie Test();

//但是我们可以自己执行迭代器函数内容 [[22 迭代器#^529961]]
ie.MoveNext(); //会执行函数遇到 yield return 为止的内容
Print(ie.Current); //得到 yield return 返回的内容
ie.MoveNext(); //执行第二次执行的内容
Print(ie.Current); // 2 yield return返回的内容
ie.MoveNext(); //执行第三次执行的内容
Print(ie.Current); // 2 yield return返回的内容
ie.MoveNext(); //执行第四次执行的内容
TestClass tc = ie.Current as TestClass;// TestClass yield return返回的内容

while(ie.MoveNext) { } //一次执行完 ie.MoveNext会返回一个bool值 表示还能不能继续执行


//2 协程调度器
//继承MonoBehavior后开启协程
//相当于是把一个协程函数（迭代器）放入unity的协程调度器中帮助我们管理进行执行
//具体的yieldreturn后面的规则也是unity定义的一些规则 [[07 协同程序#^87cc30]] 
```

**总结：可以简化理解迭代器函数**
- C#看到迭代器函数和yield return 语法糖，就会把原本是一个的函数变成"几部分"
- 我们可以通过迭代器从上到下遍历这“几部分"进行执行，就达到了将一个函数中的逻辑分时执行的目的
- 而协程调度器就是利用迭代器函数返回的内容，来进行之后的处理
- 比如unity中的协程调度器，根据yieldreturn返回的内容决定了下一次在何时继续执行迭代器函数中的"下一部分"
- 理论上来说我们可以利用迭代器函数的特点自己实现协程调度器来取代unity自带的调度器，但是一般情况下不会自己实现

## 练习

>  1. 请不使用Unity自带的协程协调器开启协程，通过迭代器函数实现每隔一秒执行函数中的一部分逻辑

```csharp title:"管理器"
//[[07 协同程序#^322e2e]] 
//数据信息类
public class YieldReturn
{
    public IEnumerator ie;
    public float time;
}
public class Coroutine_Mgr : MonoBehaviour
{
    private static Coroutine_Mgr instance;
    public static Coroutine_Mgr Instance => instance;
    private List<YieldReturn> list = new List<YieldReturn>();
    //声明一个自定义的 协程协调器
    //使协程按照我们自定义的规则执行
    public void MyStartCoroutine(IEnumerator ie)
    {
        //如果还有下一步
        if (ie.MoveNext())
        {
            //如果返回值是int
            if (ie.Current is int)
            {
                YieldReturn info = new YieldReturn();
                //存储当前的迭代器函数
                info.ie = ie;
                //存储下一步执行的时间 编译器运行时间 + 返回值设定的时间
                info.time = Time.time + (int)ie.Current;
                list.Add(info);
            }
        }
    }
    void Awake()
    {
        instance = this;
    }
    void Update()
    {
        //用Unity给我们提供的死循环 检测是否达到条件 继续执行分布函数
        //反向遍历 可以避免 在移除中间下标对应的数据时 继续遍历漏过下一个下标对应的数据
        for (int i = list.Count - 1; i >= 0; i--) 
        {
            //如果数据中记录的时间小于等于当前时间 就说明条件达成 需要继续执行分布函数
            if (list[i].time <= Time.time)
            {
                if (list[i].ie.MoveNext())
                {
                    //如果返回值是int
                    if (list[i].ie.Current is int)
                        //存储下一步执行的时间 编译器运行时间 + 返回值设定的时间
                        list[i].time = Time.time + (int)list[i].ie.Current;
                    //如果返回值不是int 就不该存在这个List中 应该根据类型放进别的List中进行处理
                    else 
                        list.RemoveAt(i);
                }
                //如果没有下一个函数了 就证明已经执行完毕逻辑 移除当前数据
                else
                    list.RemoveAt(i);
            }
        }
    }
}
```
