## 场景同步切换的缺点

- 在切换场景时，Unity会删除当前场景上所有对象，并且去加载下一个场景的相关信息
- 如果当前场景对象过多或者下一个场景对象过多，这个过程会非常的耗时会让玩家感受到卡顿

```csharp
SceneManager.LoadScene("SceneName");
```

---

## 语法 / 用法

**注意：通过协程异步加载场景时**
- 会把当前场景上没有特别处理的对象，都删除了
- 所以协程中的部分逻辑可能是执行不了的

**解决思路：** 让处理场景加载的脚本依附的对象过场景时不被移除

```csharp
//场景异步加载和资源异步加载 几乎一致 有两种方式
//1 通过 事件回调函数 异步加载
AsyncOperation ao = SceneManager.LoadSceneAsyne("SceneName"); 
//场景异步加载完毕后 就会自动调用该事件函数 如果要在加载结束后做一些事情 就可以在该函函数中写逻辑
ao.completed += (a) => { Print("加载结束"); };//场景加载完毕后可以正常执行里面的逻辑

//2 通过 协程 异步加载
IEnumerator LoadScene(string name)
{
	//异步加载场景
	AsyncOperation ao = SceneManager.LoadSceneAsyne(name); 
	//协程的优点 是异步加载结束后可以处理一些其他逻辑
	
	yield return ao;
	//加载完毕后 执行逻辑
	Print("异步加载结束");//当场景加载完毕后 这里的逻辑可能是加载不了的 因为对象都被删除了 这是因为Unity中协程的特性 脚本和对象被销毁之后 就无法执行
	//解决方法
	DontDestroyOnLoad(GameObject);
	
	//可以处理一些逻辑 比如 更新进度条
	//第一种 利用场景异步加载的进度去更新 但是不是特别准确 一般也不会直接用
	while (ao.isDone)
	{
		Print("ao.Progress");
		yield return null;
	}
	//离开循环后 可以认为加载完毕 把进度条顶满 然后隐藏
	
	//第二种 根据游戏的规则 自己定义 进度条变化的条件 根据加载
	yield return ao;
	//场景加载结束 更新20%进度
	//接着去加载场景中的其他信息 
	//比如 动态加载怪物 动态加载物品 更新到40%进度
	//动态加载 场景模型 加载结束 进度条顶满 隐藏进度条
}

StartCoroutine(LoadScene("SceneName"));
```

## 练习

>  1. 请写一个简单的场景管理器，提供统一的方法给外部用于场景异步切换外部可以传入委托用于当异步切换结束时执行某些逻辑

```csharp
public class Scene_Mgr
{
    private static Scene_Mgr instance = new Scene_Mgr();
    public static Scene_Mgr Instance => instance;

    public void LoadScene1(string name, UnityAction callBack)
    {
        AsyncOperation ao = SceneManager.LoadSceneAsync(name);
        ao.completed += (a) => { callBack(); }; 
    }
}
public class Scene_Test : MonoBehaviour
{
    void Start()
    {
        Scene_Mgr.Instance.LoadScene("TestScene", () => { print("场景加载结束"); });
    }
}
```
