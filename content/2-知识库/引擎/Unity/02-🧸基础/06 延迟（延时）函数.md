## 核心理解

> 会延时执行的函数

- 我们可以自己设定延时要执行的函数和具体延时的时间
- 是MonoBehaviour基类中实现好的方法

**我的理解：** MonoBehaviour 中的一个方法，指定一个函数过一会再执行

## 语法 / 用法

---

### 延迟函数 Invoke

**注意：**
- 延时函数第一个参数传入的是函数名字符串，找不到会弹出一个提示
- 延时函数没办法传入参数，如果函数是有参函数会弹出一个提示
- 延时函数会优先调用无参的重载函数
- 如果硬要执行有参函数，可以在无参函数里面封装、传入参数
- 函数名必须是该脚本上申明的函数，调用其他脚本的函数同样需要封装

```csharp
Invoke("函数名", 延迟时间);
Invoke("Test", 5);
```

---

### 延迟重复执行函数 InvokeRepeating

**注意事项和 Invoke 一样**

```csharp
InvokeRepeating("函数名", 第一次执行的延迟时间, 后面每一次执行的间隔时间);
InvokeRepeating("Test", 2, 1);
```

---

### 取消延迟函数 CancelInvoke

**注意：**
- 如果指定取消延迟函数，不管该函数之前被开启了多少次，延迟执行都会被同一取消

```csharp
//1 取消该脚本所有的延迟函数执行
CancelInvoke();

//2 指定函数名取消
CancelInvoke("函数名");
CancelInvoke("Test");
```

---

### 判断是否有延迟函数 IsInvoking

```csharp
//不指定函数
if (IsInvoking())
	print("存在延迟函数")
	
//指定函数	
if (IsInvoking("函数名"))
	print("存在延迟函数")
```

---

## 延迟函数受对象失活销毁影响

- 脚本或脚本依附对象失活，延迟函数可以继续执行
- 脚本或脚本依附对象销毁移除，延迟函数无法继续执行

**在生命周期函数中的运用**

```csharp
void OnEnable
{
	对象激活 的生命周期函数中 开启延迟	
}
void OnDisable
{
	对象失活 的生命周期函数中 停止延迟
}
```

## 什么时候用

> 适用场景，帮助建立使用直觉

- 在对象激活的生命周期内重复执行某个函数

## 练习

> 1. 利用延时函数实现一个计秒器

```csharp
public class Invoke_Time : MonoBehaviour
{
    public int time;
    private void Start()
    {
        InvokeRepeating("Times1", 0, 1);//方法1
        Times2();//方法2
    }


    private void Times1() { ++time; print(time + "秒"); }
    private void Times2() { ++time; print(time + "秒"); Invoke("Times2", 1); }
}
```

> 2. 请用两种方式延时销毁一个指定对象

```csharp
public class Invoke_Disable : MonoBehaviour
{
    private void Start()
    {
        Invoke("Disable", 1);//方法1 Invoke
        Destroy(GetComponent<Invoke_Disable>().gameObject, 1);//方法2 Destroy
    }
    private void Disable() { Destroy(GetComponent<Invoke_Disable>().gameObject); }
}
```

## API 速查

- Invoke （方法 延时执行函数）
- InvokeRepeating （方法 延时重复函数）
- CancelInvoke （方法 停止延时函数）
- IsInvoking （方法 是否有延迟函数待执行）
