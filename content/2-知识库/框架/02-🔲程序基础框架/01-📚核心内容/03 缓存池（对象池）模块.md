## 问题

1. 对象的频繁创建
	- 频繁的实例化对象会带来一定的性能开销

2. 对象的频繁销毁
	- 对象的频繁销毁会造成大量的内存垃圾，会造成GC的频繁触发，GC的触发，内存的释放，可能会带阿里卡顿感，影响玩家体验

---

## 主要作用

**优化资源管理、提高程序性能**
通过重复利用已经创建的对象，避免频繁的创建和销毁，从而减少系统的内存分配和垃圾回收带来的开销

---

## 基本原理

用一个“柜子”中的“各种抽屉”来装“东西”
用时去拿（没有就创造，存在就获取）
不用就还（将“东西”分门别类的放入“抽屉”中）

## 实现缓存池(对象池)模块

1. 创建PoolMgr继承 不继承MonoBehaviour的单例模式基类
2. 声明柜子(Dictionary)和抽屉(List、Stack、Queue等)容器
3. 拿东西方法
	- 有抽屉并且抽屉里有东西 直接获取
	- 没有抽屉或者抽屉里没东西 创造
4. 放东西方法
	- 有抽屉，直接放
	- 没抽屉，创建抽屉，再放
5. 清空柜子方法
	- 在切场景时，对象都会被移除，这时应该清空柜子
	- 否则会出现内存泄漏，并且下次取东西会出问题

```csharp title:缓存池模块 
/// <summary>
/// 缓存池模块
/// </summary>
public class PoolMgr : BaseManager<PoolMgr>
{
    private Dictionary<string, Stack<GameObject>> poolDic = new Dictionary<string, Stack<GameObject>>();
    private PoolMgr() { }
    /// <summary>
    /// 从缓存池取出对象
    /// </summary>
    /// <param name="name">抽屉名</param>
    public GameObject GetObj(string name)
    {
        GameObject obj;
        if (poolDic.ContainsKey(name) && poolDic[name].Count > 0) //有抽屉 并且抽屉不为空才取
        {
            obj = poolDic[name].Pop();
            obj.SetActive(true); //取出后激活已经失活的对象
        }
        else //否则实例化
        {
            obj = GameObject.Instantiate(Resources.Load<GameObject>(name));
            obj.name = name;//去掉（Clone）
        }
        return obj;
    }
    /// <summary>
    /// 往缓存池放入对象
    /// </summary>
    public void PushObj(GameObject obj) 
    {
        //目的是把对象隐藏起来
        //除了这种方式 也可以把对象放到摄像机实现外的地方 能实现目的就可以
        obj.SetActive(false); //并不是直接移除对象 而是先失活 用的时候再激活
        if (!poolDic.ContainsKey(obj.name)) //没有抽屉 创建抽屉
            poolDic.Add(obj.name, new Stack<GameObject>());
        poolDic[obj.name].Push(obj);//往抽屉放入对象
    }
    /// <summary>
    /// 清空抽屉 
    /// 主要是在切换场景时使用
    /// </summary>
    public void ClearPool() 
    {
        poolDic.Clear();
    }
}
```

### 窗口布局优化

现在直接失活对象，当之后项目做大了，抽屉多了，对象多了
游戏中成百上千个对象，在开发测试时不方便从Hierarchy窗口中查看对象获取信息
因此可以优化一下Hierarchy窗口中的布局
将对象和抽屉的关系可视化

**制作思路：**
1. 柜子管理自己的柜子根物体
2. 抽屉管理自己的抽屉根物体
3. 失活时建立父子关系，激活活时断开父子关系

**具体实现:**
1. 先实现将所有对象放入柜子根物体中
2. 再实现将对象放入对应的抽屉根物体中
	- 用面向对象的思想将抽屉相关数据行为封装起来

**将其变为可控制开启的功能**
- 这样可以避免在真正运行时，由于父子关系的频繁变换，带来一些额外的性能开销


```csharp
public class PoolData
{
    private Stack<GameObject> dataStack = new Stack<GameObject>();
    private GameObject poolObj;
    public static bool isOpenLayout = false; //是否开启窗口整理功能的标识 手动设置即可开关功能
    public int Count => dataStack.Count;
    /// <summary>
    /// 构造函数
    /// </summary>
    /// <param name="root">缓存池的根对象</param>
    /// <param name="name">缓存池的根对象的名字</param>
    public PoolData(GameObject root, string name) 
    {
        if (isOpenLayout)
        {
            poolObj = new GameObject(name); //创建柜子的根对象
            poolObj.transform.SetParent(root.transform);//把柜子的根对象设置为 缓存池对象
        }
    }
    /// <summary>
    /// 从抽屉中弹出数据对象
    /// </summary>
    /// <returns></returns>
    public GameObject Pop() 
    {
        GameObject obj = dataStack.Pop();
        obj.SetActive(true); //取出后激活已经失活的对象
        if (isOpenLayout)
            obj.transform.SetParent(null); //取消父对象
        return obj;
    }
    public void Push(GameObject obj)
    {
        obj.SetActive(false); //并不是直接移除对象 而是先失活 用的时候再激活
        if (isOpenLayout)
            obj.transform.SetParent(poolObj.transform); //设置父对象 为缓存池根对象
        dataStack.Push(obj);//通过栈记录对象数据
    }
}

        }
        else //否则实例化
        {
            obj = GameObject.Instantiate(Resources.Load<GameObject>(name));
            obj.name = name;//去掉（Clone）
        }
        return obj;
    }
    /// <summary>
    /// 往缓存池放入对象
    /// </summary>
    public void PushObj(GameObject obj) 
    {
        if (poolObj == null && PoolData.isOpenLayout) //创建缓存池根对象
            poolObj = new GameObject("Pool");
        //目的是把对象隐藏起来
        //除了这种方式 也可以把对象放到摄像机实现外的地方 能实现目的就可以
        
        if (!poolDic.ContainsKey(obj.name)) //没有抽屉 创建抽屉
            poolDic.Add(obj.name, new PoolData(poolObj, obj.name));
        poolDic[obj.name].Push(obj);//往抽屉放入对象
    }
    /// <summary>
    /// 清空抽屉 
    /// 主要是在切换场景时使用
    /// </summary>
    public void ClearPool() 
    {
        poolDic.Clear();
        poolObj = null;
    }
}
```

### 对象上限优化

目前制作的缓存池模块
理论上来说，当动态创建的对象长时间不放回抽屉
每次从缓存池中动态获取对象时，会不停的新建对象
那么也就是对象的数量是没有上限的
场景上的某种对象可以存在n个

而对象上限优化指的就是
可以控制对象数量有上限
对于不重要的资源我们没必要让其无限加量
而是将“使用最久”的资源直接抢来用

**主要目的：**
1. 更加彻底的复用资源
2. 对对象的数量上限加以限制
3. 可以优化内存空间，甚至优化性能（减少数量上限，可以减小渲染压力）

**制作思路：**
1. 在抽屉里声明一个容器用来记录正在使用的资源
2. 每次获取对象时，传入一个抽屉最大容量值（可以给一个默认值）
3. 从缓存池中获取对象时就需要创建抽屉，用于记录当前使用着的对象
4. 每次取对象时应该分情况考虑
	  - 情况1：没有抽屉时
	  - 情况2：有抽屉，并且抽屉里有没用的对象或者使用中对象超过上限时
	  - 情况3：有抽屉，但是抽屉里没有对象，使用中对象也没有超过上限时
5. 每次放回对象时
	  -  由于记录了正在使用的资源，因此每次放入抽屉时还需要从记录容器中移除对象

```csharp
/// <summary>
/// 抽屉（缓存池中的数据）对象
/// </summary>
public class PoolData
{
    //用来存储抽屉中的对象 记录的是没有使用的对象
    private Stack<GameObject> dataStack = new Stack<GameObject>();
    //记录正在使用的对象
    private List<GameObject> usedList = new List<GameObject>();
    private GameObject rootObj; //抽屉的根对象
    public int Count => dataStack.Count;
    public int UsedCount => usedList.Count;
    /// <summary>
    /// 构造函数
    /// </summary>
    /// <param name="root">缓存池的根对象</param>
    /// <param name="name">缓存池的根对象的名字</param>
    /// <param name="usedObj">实例化出的对象</param>
    public PoolData(GameObject root, string name, GameObject usedObj)
    {
        if (PoolMgr.isOpenLayout)
        {
            rootObj = new GameObject(name); //创建柜子的根对象
            rootObj.transform.SetParent(root.transform);//把柜子的根对象设置为 缓存池对象
        }
        PushUsedObj(usedObj); //把对象 记录到记录使用中的对象的容器中
    }
    /// <summary>
    /// 从抽屉中弹出数据对象
    /// </summary>
    /// <returns></returns>
    public GameObject Pop()
    {
        GameObject obj;
        if (Count > 0) //如果容器大于0 直接拿出去用
        {
            obj = dataStack.Pop();
            usedList.Add(obj);//存入到记录使用中的对象的容器中
        }
        else //否则复用场景中使用最久的对象
        {
            obj = usedList[0]; //取出索引中的第一个
            usedList.RemoveAt(0); //移除
            usedList.Add(obj); //再放入
        }
        obj.SetActive(true); //取出后激活已经失活的对象
        if (PoolMgr.isOpenLayout)
            obj.transform.SetParent(null); //取消父对象
        return obj;
    }
    /// <summary>
    /// 往抽屉中存入数据对象
    /// </summary>
    public void Push(GameObject obj)
    {
        obj.SetActive(false); //并不是直接移除对象 而是先失活 用的时候再激活
        if (PoolMgr.isOpenLayout)
            obj.transform.SetParent(rootObj.transform); //设置父对象 为缓存池根对象
        dataStack.Push(obj);//通过栈记录对象数据
        usedList.Remove(obj);//从使用中的对象容器中移除对象
    }
    /// <summary>
    /// 往记录正在使用的对象容器中存入对象
    /// </summary>
    public void PushUsedObj(GameObject obj)
    {
        usedList.Add(obj);
    }
}

/// 缓存池模块
/// </summary>
public class PoolMgr : BaseManager<PoolMgr>
{
    private Dictionary<string, PoolData> poolDic = new Dictionary<string, PoolData>();
    private GameObject rootObj; //缓存池根对象
    public static bool isOpenLayout = true; //是否开启窗口整理功能的标识 手动设置即可开关功能
    private PoolMgr() { }
    /// <summary>
    /// 从缓存池取出对象
    /// </summary>
    /// <param name="name">抽屉名</param>
    /// /// <param name="maxNum">抽屉最大容量值</param>
    public GameObject GetObj(string name, int maxNum = 100)
    {
        if (rootObj == null && isOpenLayout) //创建缓存池根对象
            rootObj = new GameObject("Pool");
        GameObject obj;
        if (!poolDic.ContainsKey(name) ||
            (poolDic[name].Count == 0 && poolDic[name].UsedCount < maxNum))//没有抽屉时
        {
            obj = GameObject.Instantiate(Resources.Load<GameObject>(name));
            obj.name = name;//去掉（Clone）
            if (!poolDic.ContainsKey(name)) //没有抽屉创建抽屉
                poolDic.Add(name, new PoolData(rootObj, name, obj));
            else
                poolDic[name].PushUsedObj(obj);//记录实例化出来的对象
        }
        //有抽屉并且抽屉中有存在的对象 或者使用的对象超过上限时
        else
            obj = poolDic[name].Pop();
        //有抽屉抽屉里面没有对象 使用的对象也没有没有超过上限时
        return obj;
    }
    /// <summary>
    /// 往缓存池放入对象
    /// </summary>
    public void PushObj(GameObject obj)
    {
        //目的是把对象隐藏起来
        //除了这种方式 也可以把对象放到摄像机实现外的地方 能实现目的就可以

    /// 主要是在切换场景时使用
    /// </summary>

        rootObj = null;
    }
```

## 让缓存池支持不继承Mono的数据结构对象

目前实现的缓存池，主要是针对场景上的GameObject对象的
只能对场景上的对象进行缓存池功能处理

但是在游戏开发中，还会存在大量的不需要挂载到场景上的实例化对象
比如一些数据结构类、逻辑类，它们并不依附于场景上的对象，而仅仅是被引用

**举例：一个自定义数据结构类**
```csharp
TestData t = new TestData();
//当不使用它时，往往会将其置空
t = null;
//下次又要使用时，再new
t = new TestData();
```

那么对于一些频繁使用的数据结构类或逻辑类，这样做也会产生大量的垃圾
因此完全可以修改缓存池模块，让其也支持对不挂载的类对象也进行复用

**主要目的：让缓存池支持回收复用不继承MonoBehaviour（不挂载GameObject）的类对象**

**优化思路：**
1. 修改缓存池管理器PoolMgr
2. 添加一个新池子容器，专门用来记录不继承MonoBehaviour（不挂载GameObject）的类对象
3. 提供专门的方法供外部使用
	- 从池子中获取对象的方法
	- 压入池子中的方法
	- 清空池子容器的方法

**注意：**
- 由于这些自定义数据或逻辑类中可能有对其他内容的引用
- 因此需要让其有一个重置数据的方法，在压入池子时重置数据

**关键点：**
1. 池子容器父类 里式替换 父类装子类
2. 池子容器子类 泛型类 确定对象类型
3. 对象父接口，用于实现重置数据方法

```csharp
/// <summary>
/// 抽屉（缓存池中的数据）对象
/// </summary>
public class PoolData
{
    //用来存储抽屉中的对象 记录的是没有使用的对象
    private Stack<GameObject> dataStack = new Stack<GameObject>();
    //记录正在使用的对象
    private List<GameObject> usedList = new List<GameObject>();
    private GameObject rootObj; //抽屉的根对象
    public int Count => dataStack.Count;
    public int UsedCount => usedList.Count;
    /// <summary>
    /// 构造函数
    /// </summary>
    /// <param name="root">缓存池的根对象</param>
    /// <param name="name">缓存池的根对象的名字</param>
    /// <param name="usedObj">实例化出的对象</param>
    public PoolData(GameObject root, string name, GameObject usedObj)
    {
        if (PoolMgr.isOpenLayout)
        {
            rootObj = new GameObject(name); //创建柜子的根对象
            rootObj.transform.SetParent(root.transform);//把柜子的根对象设置为 缓存池对象
        }
        PushUsedObj(usedObj); //把对象 记录到记录使用中的对象的容器中
    }
    /// <summary>
    /// 从抽屉中弹出数据对象
    /// </summary>
    /// <returns></returns>
    public GameObject Pop()
    {
        GameObject obj;
        if (Count > 0) //如果容器大于0 直接拿出去用
        {
            obj = dataStack.Pop();
            usedList.Add(obj);//存入到记录使用中的对象的容器中
        }
        else //否则复用场景中使用最久的对象
        {
            obj = usedList[0]; //取出索引中的第一个
            usedList.RemoveAt(0); //移除
            usedList.Add(obj); //再放入
        }
        obj.SetActive(true); //取出后激活已经失活的对象
        if (PoolMgr.isOpenLayout)
            obj.transform.SetParent(null); //取消父对象
        return obj;
    }
    /// <summary>
    /// 往抽屉中存入数据对象
    /// </summary>
    public void Push(GameObject obj)
    {
        obj.SetActive(false); //并不是直接移除对象 而是先失活 用的时候再激活
        if (PoolMgr.isOpenLayout)
            obj.transform.SetParent(rootObj.transform); //设置父对象 为缓存池根对象
        dataStack.Push(obj);//通过栈记录对象数据
        usedList.Remove(obj);//从使用中的对象容器中移除对象
    }
    /// <summary>
    /// 往记录正在使用的对象容器中存入对象
    /// </summary>
    public void PushUsedObj(GameObject obj)
    {
        usedList.Add(obj);
    }
}
/// <summary>
/// 用于在字典中利用里氏替换原则存储子类的父对象
/// </summary>
public abstract class PoolObjectBase { }
/// <summary>
/// 存储 数据结构类 和 逻辑类（不继承Mono）的容器类
/// </summary>
/// <typeparam name="T">类型</typeparam>
public class PoolObject<T> : PoolObjectBase where T : class
{
    public Queue<T> poolObjs = new Queue<T>();
}
/// <summary>
/// 想要被复用的数据结构、逻辑类必须继承该接口 实现重置数据的方法
/// </summary>
public interface IPoolObject
{
    /// <summary>
    /// 重置数据的方法
    /// </summary>
    void ResetInfo();
}
/// <summary>
/// 缓存池模块
/// </summary>
public class PoolMgr : BaseManager<PoolMgr>
{
    //存储GameObject对象池子的字典容器
    private Dictionary<string, PoolData> poolDic = new Dictionary<string, PoolData>();
    //存储数据结构、逻辑类对象的池子的字典容器
    private Dictionary<string, PoolObjectBase> poolObjectDic = new Dictionary<string, PoolObjectBase>();
    private GameObject rootObj; //缓存池根对象
    public static bool isOpenLayout = true; //是否开启窗口整理功能的标识 手动设置即可开关功能
    private PoolMgr() { }
    /// <summary>
    /// 从缓存池取出对象
    /// </summary>
    /// <param name="name">抽屉名</param>
    /// /// <param name="maxNum">抽屉最大容量值</param>
    public GameObject GetObj(string name, int maxNum = 100)
    {
        if (rootObj == null && isOpenLayout) //创建缓存池根对象
            rootObj = new GameObject("Pool");
        GameObject obj;
        if (!poolDic.ContainsKey(name) ||
            (poolDic[name].Count == 0 && poolDic[name].UsedCount < maxNum))//没有抽屉时
        {
            obj = GameObject.Instantiate(Resources.Load<GameObject>(name));
            obj.name = name;//去掉（Clone）
            if (!poolDic.ContainsKey(name)) //没有抽屉创建抽屉
                poolDic.Add(name, new PoolData(rootObj, name, obj));
            else
                poolDic[name].PushUsedObj(obj);//记录实例化出来的对象
        }
        //有抽屉并且抽屉中有存在的对象 或者使用的对象超过上限时
        else
            obj = poolDic[name].Pop();
        //有抽屉抽屉里面没有对象 使用的对象也没有没有超过上限时
        return obj;
    }
    /// <summary>
    /// 从缓存池取出对象（数据结构、逻辑类）
    /// </summary>
    /// <typeparam name="T">类型</typeparam>
    /// <param name="nameSpace">命名空间 默认为空</param>
    /// <returns></returns>
    public T GetObj<T>(string nameSpace = "") where T : class, IPoolObject, new()
    {
        string poolName = nameSpace + "_" + typeof(T).Name;//池子的名字根据命名空间和类名决定
        //有池子
        if (poolObjectDic.ContainsKey(poolName))
        {
            PoolObject<T> pool = poolObjectDic[poolName] as PoolObject<T>; //取出池子
            if (pool.poolObjs.Count > 0)//如果有可以复用的内容
            {
                Debug.Log("从池子中取出，容量为" + pool.poolObjs.Count);
                T obj = pool.poolObjs.Dequeue() as T; //取出复用
                return obj;
            }
            else //如果池子是空的
            {
                Debug.Log("从池子中new");
                T obj = new T();
                return obj;
            }
        }
        else//没池子
        {
            Debug.Log("没有池子，创建池子");
            T obj = new T();
            return obj;
        }
    }
    /// <summary>
    /// 往缓存池放入对象
    /// </summary>
    public void PushObj(GameObject obj)
    {
        //目的是把对象隐藏起来
        //除了这种方式 也可以把对象放到摄像机实现外的地方 能实现目的就可以
        poolDic[obj.name].Push(obj);//往抽屉放入对象
    }
    /// <summary>
    /// 往缓存池放入对象（数据结构、逻辑类）
    /// </summary>
    /// <typeparam name="T">类型</typeparam>
    /// <param name="obj">数据对象</param>
    /// <param name="nameSpace">命名空间 默认为空</param>
    public void PushObj<T>(T obj, string nameSpace = "") where T : class, IPoolObject, new()
    {
        if (obj == null) //如果传入的obj为空 不执行后面逻辑
            return;
        string poolName = nameSpace + "_" + typeof(T).Name;//池子的名字根据命名空间和类名决定
        //有池子
        PoolObject<T> pool;
        if (poolObjectDic.ContainsKey(poolName))
            pool = poolObjectDic[poolName] as PoolObject<T>;//取出池子 压入对象
        else//没池子
        {
            pool = new PoolObject<T>();//new一个池子
            poolObjectDic.Add(poolName, pool);//存入字典容器
        }
        obj.ResetInfo();//放入池子前 重置数据
        pool.poolObjs.Enqueue(obj);
        Debug.Log("放回池子，容量为：" + pool.poolObjs.Count);
    }
    /// <summary>
    /// 清空抽屉 
    /// 主要是在切换场景时使用
    /// </summary>
    public void ClearPool()
    {
        poolDic.Clear();
        poolObjectDic.Clear();
        rootObj = null;
    }
}
```
