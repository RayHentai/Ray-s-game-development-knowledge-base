## Resources 资源加载模块

---

### 核心理解

1. 资源加载模块的主要作用 
	- 方便统一管理各种资源加载卸载方式（Resources、AB包、UnityWebRequest、Editor） 
	- 方便异步加载资源（ Resources、AB包、UnityWebRequest ）

2. 资源加载模块的基本原理 
	- 将各资源加载模式模块化 
	- 方便在加载资源时根据自己的需求进行选择性使用

---

### 主要作用

**问题：** 
1. 实际开发中，可能会采用多种资源加载模式组合使用 逻辑分散时，不方便管理 
2. 异步加载时，往往会使用协同程序，异步加载资源会产生很多 协同程序相关的冗余代码

### 基本原理

**将各资源加载模式模块化**
比如Resources资源管理器、AB包资源管理器等等 

**让他们分别管理不同加载方式加载的资源，最终再将他们整合在一起**
方便在加载资源时根据自己的需求进行选择性使用

---

### 基本实现

1. 创建ResourcesMgr 继承 不继承MonoBehaviour的单例模式基类
2. 为它封装Resources异步加载资源的相关方法（主要目的 避免异步加载的代码冗余）
3. 为它封装Resources同步加载资源的相关方法（顺便封装）
4. 为它封装资源卸载相关方法（顺便封装）

```csharp title:资源加载模块
public class ResMgr : BaseManager<ResMgr>
{
    private ResMgr() { }
    /// <summary>
    /// 同步加载资源
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <returns></returns>
    public T Load<T>(string path) where T : UnityEngine.Object
    {
        return Resources.Load<T>(path);
    }
    /// <summary>
    /// 异步加载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    public void LoadAsync<T>(string path, UnityAction<T> callback) where T : UnityEngine.Object
    {
        //开启协程
        MonoMgr.Instance.StartCoroutine(ReallyLoadAsync<T>(path, callback));
    }
    /// <summary>
    /// 真正异步加载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    /// <returns></returns>
    private IEnumerator ReallyLoadAsync<T>(string path, UnityAction<T> callback) where T : UnityEngine.Object
    {
        ResourceRequest rq = Resources.LoadAsync<T>(path);
        yield return rq;
        callback(rq.asset as T); //加载完毕后 将资源转化为对应的资源类型 作为参数传出
    }

    /// <summary>
    /// 异步加载资源（Type重载）
    /// </summary>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    public void LoadAsync(string path, Type type, UnityAction<UnityEngine.Object> callback)
    {
        //开启协程
        MonoMgr.Instance.StartCoroutine(ReallyLoadAsync(path, type, callback));
    }
    /// <summary>
    /// 真正异步加载资源（Type重载）
    /// </summary>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    /// <returns></returns>
    private IEnumerator ReallyLoadAsync(string path, Type type, UnityAction<UnityEngine.Object> callback)
    {
        ResourceRequest rq = Resources.LoadAsync(path, type);
        yield return rq; 
        callback(rq.asset); //加载完毕后 将资源转化为对应的资源类型 作为参数传出
    }
    /// <summary>
    /// 指定卸载一个资源
    /// </summary>
    /// <param name="assetToUnload">资源对象</param>
    public void UnloadAsset(UnityEngine.Object assetToUnload)
    {
        Resources.UnloadAsset(assetToUnload);
    }
    /// <summary>
    /// 异步卸载没有使用的Resources资源
    /// </summary>
    /// <param name="callback">回调函数</param>
    public void UnloadUnusedAssets(UnityAction callback)
    {
        MonoMgr.Instance.StartCoroutine(ReallyUnloadUnusedAssets(callback));
    }
    /// <summary>
    /// 真正异步卸载没有使用的Resources资源
    /// </summary>
    /// <param name="callback">回调函数</param>
    /// <returns></returns>
    private IEnumerator ReallyUnloadUnusedAssets(UnityAction callback)
    {
        AsyncOperation ao = Resources.UnloadUnusedAssets();
        yield return ao;
        callback?.Invoke(); //卸载完毕后通知外部 执行回调函数
    }
}
```

**潜在问题：如果同时加载两次一个相同的资源，会执行两次协程**

---

### 优化

---

#### 异步加载问题

**问题：性能浪费**
Resources加载一次资源过后，该资源就一直存放在内存中作为缓存
第二次加载时发现缓存中存在该资源，会直接取出来进行使用
所以 **多次重复加载不会浪费内存**
但是 **会浪费性能**（每次加载都会去查找取出，始终伴随一些性能消耗）

每次进行异步加载时，都会开启一个协同程序
虽然Resources资源会在内部进行缓存，加载已加载过的资源，性能消耗不会太大
但是**每次开启协程的过程也会浪费性能**

因此可以优化上面的ResMgr
**不依赖Resources内部的缓存机制，而是自己来管理已经加载过的资源**
从而解决异步加载时协同程序的频繁开启造成的性能浪费

**目的：**
- 通过一个字典记录已经加载过的资源
- 每次在进行资源加载时，如果发现是已经加载过的资源，直接使用即可

**思路：**
1. 字典容器结构设计
	- 主要考虑点
		- key - 资源名（路径 + 类型 拼接而成）
		- value - 自定义数据结构类：资源、委托、协程对象等

2. 修改异步加载相关逻辑
	- 字典中不存在资源记录时
		- 开启协同程序进行加载，并且此时就要记录进字典中（这样可以避免重复异步加载）
	- 字典中存在资源记录时
		- 资源还没加载完 —— 记录委托
		- 资源已经加载完 —— 直接使用

3. 修改同步加载相关逻辑
	- 字典中不存在资源记录时
		- 直接同步加载资源记录即可
	- 字典中存在资源记录时
		- 资源还没加载完 —— 停止协程
		- 资源已经加载完 —— 直接使用

4. 修改卸载资源相关逻辑
	- 字典中存在资源记录时
		- 资源还没加载完 —— 记录删除标识，待加载完后真正移除 或者 停止协程，并且移除 
		- 资源已经加载完 —— 直接卸载，并且移除字典中资源记录

```csharp
/// <summary>
/// 资源信息基类
/// 用于 里氏替换原则 父类装子类的父类
/// </summary>
public abstract class ResInfoBase
{
    public int refCount; //引用计数
}
/// <summary>
/// 异步加载资源信息对象
/// 用于存储资源信息、异步加载委托信息、异步加载的协程信息
/// </summary>
/// <typeparam name="T">资源</typeparam>
public class ResInfo<T> : ResInfoBase
{
    public T asset; //资源
    public UnityAction<T> callback; //异步加载结束后 传递给外部的回调函数
    public Coroutine coroutine; //存储异步加载时开启的协同程序
    public bool isDel; //决定引用计数为0时 是否真正需要移除

    public void AddRefCount()
    {
        ++refCount;
    }
    public void SubRefCount()
    {
        --refCount;
        if (refCount < 0)
            Debug.LogError("当前引用计数小于0, 请检查使用和卸载是否配对执行");
    }
}

public class ResMgr : BaseManager<ResMgr>
{
    private ResMgr() { }
    private Dictionary<string, ResInfoBase> resDic = new Dictionary<string, ResInfoBase>();//用于记录加载资源的字典容器
    /// <summary>
    /// 同步加载资源
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <returns></returns>
    public T Load<T>(string path) where T : UnityEngine.Object
    {
        ResInfo<T> resInfo;
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型
        if (!resDic.ContainsKey(resName)) //如果字典中不存在资源
        {
            T res = Resources.Load<T>(path); //同步加载
            resInfo = new ResInfo<T>();
            resInfo.asset = res; //记录加载完的资源
            resInfo.AddRefCount(); //引用计数 +1
            resDic.Add(resName, resInfo); //存入字典
            return res; //返回
        }
        else //如果字典中存在资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出字典中的记录
            resInfo.AddRefCount(); //引用计数 +1
            if (resInfo.asset == null) //如果资源没有加载完毕
            {
                MonoMgr.Instance.StopCoroutine(resInfo.coroutine); //停止协程
                T res = Resources.Load<T>(path); //同步加载
                resInfo.asset = res; //记录加载完的资源
                resInfo.callback?.Invoke(res); //执行还未执行的回调函数
                //清除无用引用
                resInfo.callback = null;
                resInfo.coroutine = null;
                return res;
            }
            else //如果资源加载完毕
                return resInfo.asset; //直接返回资源
        }
    }
    /// <summary>
    /// 异步加载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    public void LoadAsync<T>(string path, UnityAction<T> callback) where T : UnityEngine.Object
    {
        ResInfo<T> resInfo;
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型
        if (!resDic.ContainsKey(resName)) //如果字典中没有记录该资源
        {
            resInfo = new ResInfo<T>(); //声明一个资源信息对象
            resInfo.AddRefCount(); //引用计数 +1
            resDic.Add(resName, resInfo);
            resInfo.callback += callback; //记录传入的回调函数 等资源加载完了再使用 
            //开启协程 进行异步加载 并且记录协同程序 用于之后可能的停止
            resInfo.coroutine = MonoMgr.Instance.StartCoroutine(ReallyLoadAsync<T>(path));
        }
        else //如果字典中记录了该资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出资源信息
            resInfo.AddRefCount(); //引用计数 +1
            if (resInfo.asset == null) //如果没有加载完毕
                resInfo.callback += callback; //叠加回调函数
            else //如果加载完毕
                callback?.Invoke(resInfo.asset); //调用回调函数
        }
    }
    /// <summary>
    /// 真正异步加载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <returns></returns>
    private IEnumerator ReallyLoadAsync<T>(string path) where T : UnityEngine.Object
    {
        ResInfo<T> resInfo;
        ResourceRequest rq = Resources.LoadAsync<T>(path);
        yield return rq;
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型
        if (resDic.ContainsKey(resName)) //判断是否存在该资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出资源信息
            resInfo.asset = rq.asset as T; //记录加载完毕的资源
            if (resInfo.refCount == 0) //如果引用计数为0
                UnloadAsset<T>(path, resInfo.isDel, false, null); //卸载资源
            else //否则继续执行逻辑
            {
                resInfo.callback?.Invoke(resInfo.asset); //将加载完毕的资源传递出去
                //资源加载完毕
                resInfo.callback = null; //将回调函数置空
                resInfo.coroutine = null;//将协程信息置空
            }
        }
    }

    /// <summary>
    /// 异步加载资源（Type重载）
    /// </summary>
    /// <param name="path">资源路径</param>
    /// <param name="type">资源类型</param>
    /// <param name="callback">回调函数</param>
    [Obsolete("注意：建议使用泛型加载方式，如果实在要用Type加载，一定不能和泛型加载混用去加载同类型同名资源")]
    public void LoadAsync(string path, Type type, UnityAction<UnityEngine.Object> callback)
    {
        ResInfo<UnityEngine.Object> resInfo;
        string resName = path + "_" + type.Name;//自定义的资源唯一ID：路径 + 资源类型
        if (!resDic.ContainsKey(resName)) //如果字典中没有记录该资源
        {
            resInfo = new ResInfo<UnityEngine.Object>(); //声明一个资源信息对象
            resInfo.AddRefCount(); //引用计数 +1
            resDic.Add(resName, resInfo);
            resInfo.callback += callback; //记录传入的回调函数 等资源加载完了再使用 
            //开启协程 进行异步加载 并且记录协同程序 用于之后可能的停止
            resInfo.coroutine = MonoMgr.Instance.StartCoroutine(ReallyLoadAsync(path, type));
        }
        else //如果字典中记录了该资源
        {
            resInfo = resDic[resName] as ResInfo<UnityEngine.Object>; //取出资源信息
            resInfo.AddRefCount(); //引用计数 +1
            if (resInfo.asset == null) //如果没有加载完毕
                resInfo.callback += callback; //叠加回调函数
            else //如果加载完毕
                callback?.Invoke(resInfo.asset); //调用回调函数
        }
    }
    /// <summary>
    /// 真正异步加载资源（Type重载）
    /// </summary>
    /// <param name="path">资源路径</param>
    /// <param name="type">资源类型</param>
    /// <returns></returns>
    private IEnumerator ReallyLoadAsync(string path, Type type)
    {
        ResInfo<UnityEngine.Object> resInfo;
        ResourceRequest rq = Resources.LoadAsync(path, type);
        yield return rq;
        string resName = path + "_" + type.Name;//自定义的资源唯一ID：路径 + 资源类型
        if (resDic.ContainsKey(resName)) //判断是否存在该资源
        {
            resInfo = resDic[resName] as ResInfo<UnityEngine.Object>; //取出资源信息
            resInfo.asset = rq.asset; //记录加载完毕的资源
            if (resInfo.refCount == 0) //如果引用计数为0
                UnloadAsset(path, type, resInfo.isDel, false, null); //卸载资源
            else //否则继续执行逻辑
            {
                resInfo.callback?.Invoke(resInfo.asset); //将加载完毕的资源传递出去
                //资源加载完毕
                resInfo.callback = null; //将回调函数置空
                resInfo.coroutine = null;//将协程信息置空
            }
        }
    }
    /// <summary>
    /// 异步卸载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源</typeparam>
    /// <param name="path">资源路径</param>
    /// <param name="isDel">是否立即卸载 默认false</param>
    /// <param name="isSub">是否要减少引用计数 默认true</param>
    /// <param name="callback">回调函数</param>
    public void UnloadAsset<T>(string path, bool isDel = false, bool isSub = true, UnityAction<T> callback = null)
    {
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型
        ResInfo<T> resInfo;
        if (resDic.ContainsKey(resName)) //如果存在资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出资源
            if (isSub) //如果要减
                resInfo.SubRefCount(); //引用计数 -1
            resInfo.isDel = isDel; //记录引用计数为0时 是否马上移除的标识
            if (resInfo.asset != null && resInfo.refCount == 0 && resInfo.isDel) //如果资源已经加载结束 并且引用计数为0 并且马上需要移除时
            {
                resDic.Remove(resName); //从字典移除
                Resources.UnloadAsset(resInfo.asset as UnityEngine.Object); //直接卸载
            }
            else if (resInfo.asset == null) //如果资源还没加载结束
            {
                if (callback != null)
                    //当不想使用异步加载的资源时 应该移除回调记录 而不是直接卸载资源
                    resInfo.callback -= callback;
            }
        }
    }
    /// <summary>
    /// 异步卸载资源（Type重载）
    /// </summary>
    /// <param name="path">资源路径</param>
    /// <param name="type">资源类型</param>
    /// <param name="isDel">是否立即卸载 默认false</param>
    /// <param name="isSub">是否要减少引用计数 默认true</param>
    /// <param name="callback">回调函数</param>
    public void UnloadAsset(string path, Type type, bool isDel = false, bool isSub = true, UnityAction<UnityEngine.Object> callback = null)
    {
        string resName = path + "_" + type.Name;//自定义的资源唯一ID：路径 + 资源类型
        ResInfo<UnityEngine.Object> resInfo;
        if (resDic.ContainsKey(resName)) //如果存在资源
        {
            resInfo = resDic[resName] as ResInfo<UnityEngine.Object>; //取出资源
            if (isSub) //如果要减
                resInfo.SubRefCount(); //引用计数 -1
            resInfo.isDel = isDel; //记录引用计数为0时 是否马上移除的标识
            if (resInfo.asset != null && resInfo.refCount == 0 && resInfo.isDel) //如果资源已经加载结束 并且引用计数为0 并且马上需要移除时
            {
                resDic.Remove(resName); //从字典移除
                Resources.UnloadAsset(resInfo.asset as UnityEngine.Object); //直接卸载
            }
            else if (resInfo.asset == null) //如果资源还没加载结束
            {
                if (callback != null)
                    //当不想使用异步加载的资源时 应该移除回调记录 而不是直接卸载资源
                    resInfo.callback -= callback;
            }
        }
    }
    /// <summary>
    /// 异步卸载没有使用的Resources资源
    /// </summary>
    /// <param name="callback">回调函数</param>
    public void UnloadUnusedAssets(UnityAction callback)
    {
        MonoMgr.Instance.StartCoroutine(ReallyUnloadUnusedAssets(callback));
    }
    /// <summary>
    /// 真正异步卸载没有使用的Resources资源
    /// </summary>
    /// <param name="callback">回调函数</param>
    /// <returns></returns>
    private IEnumerator ReallyUnloadUnusedAssets(UnityAction callback)
    {
        //真正移除不使用的资源之前 应该把自己记录的引用计数为0 并且没有被移除记录的资源移除掉
        List<string> list = new List<string>();
        foreach (string path in resDic.Keys)
        {
            if (resDic[path].refCount == 0) //找到所有引用计数为0的资源
                list.Add(path);
        }
        foreach (string path in list) //不能一边遍历一边移除 分步移除
            resDic.Remove(path); //移除所有引用计数为0的资源
        AsyncOperation ao = Resources.UnloadUnusedAssets();
        yield return ao;
        callback?.Invoke(); //卸载完毕后通知外部 执行回调函数
    }
    /// <summary>
    /// 移除字典

    }
    /// <summary>
    /// 真正移除字典
    /// </summary>
    /// <param name="callback">回调函数</param>
    /// <returns></returns>
    private IEnumerator ReallyClearDic(UnityAction callback)
    {
        resDic.Clear();
        AsyncOperation ao = Resources.UnloadUnusedAssets();
        yield return ao;
        callback(); //卸载完毕后 通知外部 执行回调函数
    }
    /// <summary>
    /// 得到引用计数的方法
    /// </summary>
    /// <typeparam name="T">资源</typeparam>

            return (resDic[resName] as ResInfo<T>).refCount;
        return 0;
    }
```

**潜在问题：**
1. 在卸载资源时，我们并不知道是否还有地方使用着该资源
2. UnloadUnusedAssets是卸载没有使用的资源，我们无法判断是否使用

#### 引用计数

**通过引用计数判断资源是否使用的问题**
引用计数是一种内存管理技术，用于跟踪资源被引用的次数
通过一个整形变量来记录资源的使用次数
当有对象引用该资源时，计数器会增加；当对象不再引用该资源时，计数器会减少

**思路：**
1. 为ResInfo类加入引用计数成员变量和方法
2. 使用资源时加
3. 不使用资源时减
4. 处理异步回调问题，某一个异步不使用资源了应该移除回调函数的记录
	- 场景：某次异步加载资源的地方不需要使用该资源了
5. 修改移除资源函数逻辑，引用计数为0时才真正移除资源
6. 考虑资源频繁移除问题，加入马上移除bool标签
	- 场景：避免一些大资源引用计数为0时被马上卸载，造成卡顿
7. 修改移除不使用资源函数逻辑，释放时清楚引用计数为0的记录

```csharp
/// <summary>
/// 资源信息基类
/// 用于 里氏替换原则 父类装子类的父类
/// </summary>
public abstract class ResInfoBase
{
    public int refCount; //引用计数
}
/// <summary>
/// 异步加载资源信息对象
/// 用于存储资源信息、异步加载委托信息、异步加载的协程信息
/// </summary>
/// <typeparam name="T">资源</typeparam>
public class ResInfo<T> : ResInfoBase
{
    public T asset; //资源
    public UnityAction<T> callback; //异步加载结束后 传递给外部的回调函数
    public Coroutine coroutine; //存储异步加载时开启的协同程序
    public bool isDel; //决定引用计数为0时 是否真正需要移除

    public void AddRefCount()
    {
        ++refCount;
    }
    public void SubRefCount()
    {
        --refCount;
        if (refCount < 0)
            Debug.LogError("当前引用计数小于0, 请检查使用和卸载是否配对执行");
    }
}

    /// 同步加载资源
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <returns></returns>
    public T Load<T>(string path) where T : UnityEngine.Object
    {
        ResInfo<T> resInfo;
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型
        if (!resDic.ContainsKey(resName)) //如果字典中不存在资源
        {
            T res = Resources.Load<T>(path); //同步加载
            resInfo = new ResInfo<T>();
            resInfo.asset = res; //记录加载完的资源
            resInfo.SubRefCount(); //引用计数 +1
            resDic.Add(resName, resInfo); //存入字典
            return res; //返回
        }
        else //如果字典中存在资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出字典中的记录

            resInfo.AddRefCount(); //引用计数 +1
            if (resInfo.asset == null) //如果资源没有加载完毕
            {
                MonoMgr.Instance.StopCoroutine(resInfo.coroutine); //停止协程
                T res = Resources.Load<T>(path); //同步加载
                resInfo.asset = res; //记录加载完的资源
                resInfo.callback?.Invoke(res); //执行还未执行的回调函数
                //清除无用引用
                resInfo.callback = null;
                resInfo.coroutine = null;
                return res;
            }
            else //如果资源加载完毕
                return resInfo.asset; //直接返回资源
        }
    }
    /// <summary>
    /// 异步加载资源（泛型重载）
    /// </summary>
    /// <typeparam name="T">资源类型</typeparam>
    /// <param name="path">资源路径</param>
    /// <param name="callback">加载完毕后执行的回调函数</param>
    public void LoadAsync<T>(string path, UnityAction<T> callback) where T : UnityEngine.Object
    {
        ResInfo<T> resInfo;
        string resName = path + "_" + typeof(T).Name;//自定义的资源唯一ID：路径 + 资源类型

        {

        }
        else //如果字典中记录了该资源
        {
            resInfo = resDic[resName] as ResInfo<T>; //取出资源信息
            resInfo.AddRefCount(); //引用计数 +1
            if (resInfo.asset == null) //如果没有加载完毕
                resInfo.callback += callback; //叠加回调函数
            else //如果加载完毕
                callback?.Invoke(resInfo.asset); //调用回调函数
        }
```

## Editor 资源加载模块

---

### 核心理解

1. Editor资源加载的主要作用
	- 当项目主要采用AssetBundle资源加载时
	- 用于在开发时进行资源加载，方便资源管理，避免资源重复打包

2. Editor资源加载的基本原理
	- 利用Unity编辑器拓展中的相关API来进行加载即可
		- AssetDatabase.LoadAssetAtPath
		- EditorGUIUtility.Load

---

### 主要作用

**在进行项目开发时，最终发布的项目一般为了达到以下两个目的**
1. 减小包体大小
2. 热更新

最终会通过AssetBundle资源加载作为主要加载方式
那么也就是说最终我们发布项目时，几乎所有的的游戏资源都会在AB包中的
AB包最终会放置在 StreamingAssets中 或者 远程资源服务器中

但是在开发期间，**如果进行频繁的资源打包（资源放入AssetBundle包），会大大降低我们的开发效率**

因此在开发功能时，可以不通过AB包加载资源，而是在最终要测试或发布时才把资源整合进AB包
但是为了方便资源管理，也不能将资源放在Resources中来进行加载，因为最终Resources中的资源会被打包出去。

那么此时Editor资源加载的作用就体现出来了，可以将资源放进不会被打包的Editor文件夹中，开发时通过Editor资源加载，测试和发布时使用AssetBundle资源加载

**主要原因：Editor文件夹中的内容不会被打包出去，比起放入Resources中更方便管理**

**主要作用：**
1. 避免在开发时频繁的打AB包降低开发效率
2. 方便资源管理，避免资源重复打包

### 基本原理

利用Unity编辑器拓展相关API：
1. `AssetDatabase.LoadAssetAtPath`
	- 可以加载工程路径下指定位置资源
2. `EditorGUIUtility.Load`
	- 可以加载工程路径下Editor Default Resource文件夹下资源
  
可以根据自己的需求，选择任意API进行使用

**主要使用的API：**
1. AssetDatabase.LoadAssetAtPath   用于加载单个资源
2. AssetDatabase.LoadAllAssetRepresentationsAtPath 用于加载图集资源中的内容(该API用于加载所有子资源)
**注意：这两个API加载资源的路径都是从 Assets/... 开始**

**创建用于放置资源的文件夹**
可以创建Editor文件夹，以后最终会打包为AB包的资源都放置在此处

## AssetBundle 资源加载模块 优化

---

### 重要问题
如果想要将ABMgr中的异步加载方法改为真正意义上的异步
即：不仅从AB包中加载资源是异步的，还需要在加载AB包时也采用异步

那么就需要考虑一个问题
如果当我们正在异步加载AB包时，又进行了一次同步加载AB包
是否会报错（因为AB包不允许重复加载）

通过验证得出结论
异步加载AB包时，再同步加载AB包，会因重复加载而报错
因此我们在修改逻辑时应该考虑到这个问题

在进行同步加载时如果遇到有AB包正在被异步加载应该如何解决？
在进行异步加载时再重复加载相同AB包是会报错的
即使是同步加载，我们也必须等待异步加载结束，再进行下一步

---

### 目标

将ABMgr中的异步加载方法彻底异步化
让其中的依赖包加载、资源包加载、资源加载都变为异步加载

**注意：主包可以保留同步加载**

**需要考虑的问题**
1. 某个AB包当正在异步加载时又进行重复加载
	- 遇到这种情况时，我们需要避免重复加载报错
	- 因此我们不应再次加载，而是等待之前的异步加载结束后直接使用

2. 正在加载某个AB包时
	- 卸载AB包：如果正在加载中，不允许卸载
	- 清空AB包：停止所有协同程序，在清理AB包

## 资源加载模块整合

EditorResMgr的主要目的是**方便资源管理**
避免频繁的打AB包进行测试
它主要是为AB包预备资源服务的
因此可以将它和ABMgr整合在一起

**为什么不整合ResMgr和UWQResMgr?**
这两种资源加载管理器相对比较独立
一个专门用于加载Resources下的资源
一个专门用于加载指定位置的资源
当使用它们时那一定是确定了位置和加载方式后才会使用的
