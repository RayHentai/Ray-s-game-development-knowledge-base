1. 解耦程序模块，降低程序耦合度 
2. 可以降低游戏中不同模块的耦合度 
3. 不需要直接引用或依赖于彼此的具体实现

## 基本原理

**事件中心的基本原理**
通过一个中心化的机制 使得多个系统、模块、对象之间可以进行松耦合的通信 
只需要利用C#中的字典和委托，再结合观察者设计模式的基本原理便可以制作出事件中心模块

**观察者设计模式：** 其主要原理是定义了一种一对多的依赖关系，使得当一个对象的状态发生变化时，所有依赖于它的对象都能够得到通知并自动更新

![[事件中心 原理 图示.png]]

## 基本实现

1. 创建EventCenter继承 不继承MonoBehaviour的单例模式基类
2. 声明管理事件用容器
3. 实现关键方法
  - 触发(分发)事件 方法
  - 添加事件监听者 方法
  - 移除事件监听者 方法
  - 清除所有事件监听者 方法

```csharp title:事件中心
/// <summary>
/// 事件中心模块
/// </summary>
public class EventCenter : BaseManager<EventCenter>
{
    private Dictionary<string, UnityAction> eventDic = new Dictionary<string, UnityAction>();//存储事件的容器
    private EventCenter() { }

    /// <summary>
    /// 触发事件
    /// </summary>
    /// <param name="eventName">事件名</param>
    public void EventTrigger(string eventName) 
    {
        if (eventDic.ContainsKey(eventName)) //存在事件才去执行
            eventDic[eventName]?.Invoke();
    }
    /// <summary>
    /// 添加监听者
    /// </summary>
    /// <param name="eventName">监听事件名</param>
    /// <param name="func">需要处理的事情</param>
    public void AddEventListener(string eventName, UnityAction func)
    {
        if (eventDic.ContainsKey(eventName))
            eventDic[eventName] += func;
        else
        {
            eventDic.Add(eventName, null);
            eventDic[eventName] += func; //这样处理的原因是因为 委托的坑 直接在Add中new 一个新委托无法正常移除
        }
    }
    /// <summary>
    /// 移除监听者
    /// </summary>
    /// <param name="eventName">事件名</param>
    /// <param name="func">需要处理的事情</param>
    public void RemoveEventListener(string eventName, UnityAction func)
    {
        if (eventDic.ContainsKey(eventName))
            eventDic[eventName] -= func;
    }
    /// <summary>
    /// 清空所有事件的监听
    /// </summary>
    public void Clear()
    {
        eventDic.Clear();
    }
    /// <summary>
    /// 清除指定事件的监听
    /// </summary>
    /// <param name="eventName">事件名</param>
    public void Clear(string eventName)
    {
        if (eventDic.ContainsKey(eventName))
        eventDic.Remove(eventName);
    }
}

```

### 参数传递

**希望在触发事件时传递数据，比如**
- 怪物死亡时，将怪物信息传递出去
- 获取奖励时，将奖励信息传递出去
- 等等

**传递数据的主要目的是**
- 可以在各系统、模块、对象中获取到我们希望获取的有用信息

**制作思路和具体实现**
目前触发事件，是通过执行委托中存储的函数来执行各系统对应的逻辑
那么需要传递参数，很自然的联想到应该从委托入手

但是参数的类型可能有多种多样，我们可以采用万物之父 object 利用里式替换原则
父类容器装载子类对象的方式来传递参数


### 参数传递自定义

参数传递的优化中 通过万物之父 object 利用里式替换原则（父类装子类）实现了在事件中心中传递参数
但是使用object时，当传递值类型参数时，存在装箱拆箱的性能浪费

而参数传递自定义的优化 
可以实现 在传递数据时，参数类型可以根据需求自己定义，想传什么就传什么，而不是固定的object类型

**制作思路：**
想让传递的参数类型可变，并且想传什么就传什么
可以很自然的联想到使用**泛型**来制作该功能

但是需要解决以下关键问题：**字典容器中的 泛型委托 如何提前定好类型**
如果直接让事件中心用泛型 就会破坏单例模式的唯一性

```csharp
public class EventCenter<T> : BaseManager<EventCenter<T>> 
{
	private Dictionary<string, UnityAction<T>> eventDic = new Dictionary<string, UnityAction<T>>();//存储事件的容器
}

public class Test
{
	EventCenter<string>.Instance
	EventCenter<int>.Instance //此时事件中心模块就不唯一了
}
```

解决方案：**里式替换原则（父类装子类）**

```csharp
/// <summary>
/// 用于 里氏替换原则 父类装子类的父类
/// </summary>
public abstract class EventInfoBase { }
/// <summary>
/// 用来包裹 对应观察者委托函数的类（有参函数）
/// </summary>
/// <typeparam name="T"></typeparam>
public class EventInfo<T> : EventInfoBase
{
    public UnityAction<T> actions; //真正的观察者对应的函数 记录在其中
    public EventInfo(UnityAction<T> action)
    {
        actions += action;
    }
}
/// <summary>
/// 用来包裹 对应观察者委托函数的类（无参函数）

    }
    /// <summary>
    /// 触发事件（无参）
    /// </summary>
```

### 事件名优化

**目前通过 字符串 作为事件名来区分各事件明显的缺点：**
- 若触发或监听时，事件名 字符串 拼写错误会导致无法正确监听或触发事件

**解决方案：将事件名使用枚举进行统一管理**
