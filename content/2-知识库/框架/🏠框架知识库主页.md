## 🎯 学习目标

**MVC**
- [ ] 能说清楚 MVC 的核心分层思想，为什么要分层
- [ ] 理解 MVP 和 MVC 的关键区别（View 是否持有 Model 引用）
- [ ] 理解 MVVM 的数据绑定机制
- [ ] 知道 PureMVC 的 Facade / Proxy / Mediator / Command 各自负责什么
- [ ] 能根据项目规模选出合适的架构模式并说出理由

**程序基础框架**
- [ ] 理解为什么要有框架，框架解决了什么问题（代码冗余 / 耦合度 / 可维护性）
- [ ] 能独立搭建单例基类，区分继承与不继承 MonoBehaviour 两种方案
- [ ] 理解事件中心的观察者模式原理，能解释为什么它能降低耦合
- [ ] 理解缓存池如何减少 GC，能说清楚"借"和"还"的机制
- [ ] 能串联起框架各模块之间的依赖关系

## ✅ 自检清单

### MVC
- [ ] MVC 三层各自的职责（Model 管数据、View 管显示、Controller 管逻辑）
- [ ] MVP 中 Presenter 代替 Controller，View 不直接引用 Model
- [ ] MVVM 的 ViewModel 通过双向绑定驱动视图更新
- [ ] MVE 的 Entity 层是什么，和 MVC 有何区别
- [ ] PureMVC：Facade 是唯一入口，Proxy 管数据，Mediator 管视图，Command 管指令

### 程序基础框架
- [ ] 单例基类：不继承 Mono 的潜在风险（构造函数 / 多线程），继承 Mono 的解决方案
- [ ] 公共 Mono 模块：非 Mono 脚本如何借助它使用 Update 和协程
- [ ] 缓存池：Dictionary + Queue 的"柜子-抽屉-东西"结构
- [ ] 事件中心：字典 + 委托，触发 / 监听 / 取消监听 三个操作
- [ ] 资源加载：同步 Resources vs 异步 AB 包，如何统一封装
- [ ] UI 管理：Panel 基类自动查找组件，Manager 统一管理面板显隐
- [ ] 场景切换：同步 vs 异步，如何通过事件中心传进度
- [ ] 输入控制：为什么要把 Input 抽出来，怎么配合事件中心分发
