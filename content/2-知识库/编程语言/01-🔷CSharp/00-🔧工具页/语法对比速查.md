## 类型系统

### struct vs class

| | struct | class |
|-|--------|-------|
| 存储位置 | 栈（值类型） | 堆（引用类型） |
| 赋值行为 | 拷贝值，互不影响 | 拷贝引用，共享同一对象 |
| 继承 | ❌ 不能继承/被继承 | ✅ 支持 |
| protected | ❌ 不能用 | ✅ 可以 |
| 构造函数 | 不能声明无参构造 | 可以 |
| 有参构造后无参构造 | ✅ 不会被顶掉 | ❌ 会被顶掉 |
| 适用场景 | 轻量数据（坐标/颜色） | 复杂对象（玩家/怪物） |

**关联：** [[27 结构体与类的区别]]

### abstract class vs interface

| | 抽象类 | 接口 |
|-|--------|------|
| 关键字 | `abstract class` | `interface` |
| 实例化 | ❌ 不能 | ❌ 不能 |
| 成员变量 | ✅ 可以有 | ❌ 不能有 |
| 构造函数 | ✅ 可以有 | ❌ 不能有 |
| 方法实现 | ✅ 可以有普通方法 | ❌ 只能有声明 |
| 继承数量 | 只能继承**一个** | 可以实现**多个** |
| 表达含义 | 表示**对象**（is-a） | 表示**行为**（can-do） |

**一句话区分：** 动物是抽象类（猫是动物），会飞是接口（猫不会飞但飞机会）

**关联：** [[28 抽象类与接口的区别]]

### virtual vs abstract

| | virtual | abstract |
|-|---------|---------|
| 方法体 | ✅ 有（默认实现） | ❌ 无（有大括号会报错）|
| 子类是否必须重写 | 可选 | **必须** |
| 在哪里定义 | 普通类/抽象类 | **只能在抽象类中** |
| 关键字 | `virtual` / `override` | `abstract` / `override` |

## 访问和查找

### is vs as

| | is | as |
|-|----|-----|
| 用途 | 判断类型 | 转换类型 |
| 返回值 | `bool` | 目标类型对象，失败返回 `null` |
| 失败时 | 返回 false，不崩 | 返回 null，调用方法会崩 |
| 用法 | `if (obj is Dog)` | `Dog d = obj as Dog;` |

```csharp
// 标准用法（先 is 再 as）
if (obj is Dog)
    (obj as Dog).Bark();
```

**关联：** [[15 里氏替换原则与is as]]

### == vs Equals vs ReferenceEquals

| | `==` | `Equals()` | `ReferenceEquals()` |
|-|------|-----------|-------------------|
| 值类型 | 比较值 | 比较值 | 始终 false |
| 引用类型 | 默认比较引用地址 | 默认比较引用，可以重写 | 比较引用地址 |
| string 特殊 | 比较内容 | 比较内容 | 比较引用地址 |

```csharp
string a = "hello";
string b = "hello";
a == b            // true（string 重写了 ==）
a.Equals(b)       // true
ReferenceEquals(a, b) // 不一定（取决于字符串驻留）
```

## 函数参数

### ref vs out vs 普通参数

| | 普通参数 | ref | out |
|-|---------|-----|-----|
| 传递方式 | 值拷贝（值类型）/ 引用（引用类型） | 引用 | 引用 |
| 调用前必须初始化 | 不要求 | **必须** | 不要求 |
| 函数内必须赋值 | 不要求 | 不要求 | **必须** |
| 典型用途 | 普通传参 | 让函数修改外部变量 | 函数返回多个值 |

**关联：** [[08 ref 与 out]]

### 委托 vs 事件

| | delegate | event |
|-|----------|-------|
| 本质 | 函数容器 | 委托的安全封装 |
| 类外赋值 (`=`) | ✅ 可以 | ❌ 不能 |
| 类外调用 | ✅ 可以 | ❌ 不能 |
| 类外 `+=/-=` | ✅ 可以 | ✅ 可以 |
| 防止外部乱用 | ❌ 不防 | ✅ 防 |

**一句话区分：** 委托是变量，事件是带门卫的委托。

### 匿名函数 vs Lambda

```csharp
// 匿名函数
Action<int> a = delegate(int x) { Console.WriteLine(x); };

// Lambda（等价写法）
Action<int> b = (x) => { Console.WriteLine(x); };
Action<int> c = x => Console.WriteLine(x);  // 单参数/单行可以更简
```

本质一样，Lambda 是匿名函数的语法糖，推荐用 Lambda。
唯一区别：匿名函数可以省略参数列表（`delegate { ... }`），Lambda 不行。

**关联：** [[14 匿名函数]] | [[15 Lambda 表达式]]

## 常量相关

### const vs static vs readonly

| | const | static | readonly |
|-|-------|--------|---------|
| 修改 | ❌ 不能 | ✅ 可以 | ❌ 不能（只能在构造时赋值） |
| 能修饰什么 | 只能修饰变量 | 变量/方法/属性/类 | 只能修饰字段 |
| 赋值时机 | 声明时 | 随时 | 声明时 或 构造函数中 |
| 通过类名访问 | ✅ | ✅ | ❌（需要实例） |
| 本质 | 编译期常量 | 静态成员 | 运行期只读 |

## 类型转换

### 隐式转换 vs 显式转换 vs Parse vs Convert

| 方式 | 语法 | 特点 |
|------|------|------|
| 隐式转换 | `int i = 1; long l = i;` | 自动，只能从小范围到大范围 |
| 显式转换（强转） | `int i = (int)3.14f;` | 手动，可能丢精度，不匹配会崩 |
| Parse | `int.Parse("123")` | 字符串→数值，格式错误会崩 |
| TryParse | `int.TryParse("abc", out int r)` | 安全版，失败返回 false |
| Convert | `Convert.ToInt32("123")` | 更智能，支持更多类型，null 返回 0 |
| as | `Dog d = obj as Dog;` | 引用类型转换，失败返回 null |
| (T) | `Dog d = (Dog)obj;` | 强转，失败抛 InvalidCastException |

**关联：** [[09 类型转换]] | [[15 里氏替换原则与is as]]

## 空值处理

### null vs ? vs ?? vs ?.

```csharp
// null：引用类型的空值
string s = null;

// ? 可空值类型：让值类型也能为 null
int? n = null;

// ?? 空合并：左边为 null 则取右边
string result = s ?? "默认值";

// ?. 空传播（C# 6）：左边为 null 则整个表达式返回 null，不崩
int? len = s?.Length;  // s 为 null 时返回 null 而不是崩溃
```
