## 核心理解

- 用 `static` 修饰的成员变量、方法、属性等
- 特点是直接用**类名点出使用**（比如 `Console.WriteLine`）

---

## 为什么可以直接使用

> 程序中不能无中生有，要使用的对象、变量、函数都要在内存中分配内存空间。

> 实例化对象的目的就是分配内存空间，在程序中产生一个抽象的对象，**而静态成员在程序开始运行时就会分配内存空间**，所以就可以直接使用。

---

## 静态成员与程序同生共死

> **只要使用了静态成员，直到程序结束时内存空间才会被释放。**

> 所以一个静态成员就会有自己唯一的一个内存"小房间"，这让静态成员有了**唯一性**。

> 在任何地方使用都是用的小房间里的内容，改变了它也就是改变了小房间里面的内容。

---

## 声明与使用

**注意：**
- 成员变量只能将对象实例化出来后，才能点出来使用，不能无中生有
- 在静态函数中不能直接使用非静态成员，否则会报错
- 非静态函数可以使用静态成员

```csharp
// 修饰符 和 static 的先后顺序没有限制
public static int score = 0;
static public int level = 1;

class Test
{
    public static int staticVal = 10;
    public int normalVal = 20;
	
    public static void StaticMethod()
    {
        Console.WriteLine(staticVal);   // ✅ 可以
        // Console.WriteLine(normalVal); // ❌ 报错，不知道是哪个对象的 normalVal
    }
	
    public void NormalMethod()
    {
        Console.WriteLine(staticVal);   // ✅ 可以
        Console.WriteLine(normalVal);   // ✅ 可以
    }
}



// 常用场景：
// - 常用的唯一的变量（如圆周率）
// - 方便别人获取的对象声明
// - 常用唯一的方法（如数学计算相关函数）
```

---

## 什么时候用

> 适用场景，帮助建立使用直觉

- 常用的唯一的变量（如圆周率）
- 方便别人获取的对象声明
- 常用唯一的方法（如数学计算相关函数）
- 单例模式

## 常量 vs 静态变量

|         | const        | static        |
| ------- | ------------ | ------------- |
| 必须初始化   | ✅ 必须         | ❌ 不要求         |
| 能否修改    | ❌ 不能         | ✅ 可以          |
| 能修饰什么   | 只能修饰变量       | 变量/方法/属性等     |
| 访问修饰符顺序 | const 一定写在后面 | static 没有顺序要求 |
| 本质      | 常量是特殊的静态变量   | —             |
| 相同点     | 都可以通过类名点出来使用 | —             |

## 练习

> 1. 请说出 const 和 static 的区别

**相同：**
- 都可以通过类名点出来使用

**不同：**
  - const 声明时必须赋值，且不能修改
  - const 只能修饰变量，static 可以修饰方法、属性等
  - const 不能写在访问修饰符前面，static 没有顺序要求

> 2. 请用静态成员相关知识实现：一个类对象，在整个应用程序的生命周期中有且仅会有一个该对象的存在，不能在外部实例化，直接通过类名就能得到唯一的对象 
>    [[01 单例模式基类]]

```csharp
class Test
{
    private static Test t = new Test();
    public int testInt = 999;

    public static Test T
    {
        get { return t; }
    }

    private Test() { }  // 私有构造，外部无法 new
}

Console.WriteLine(Test.T.testInt);
```
