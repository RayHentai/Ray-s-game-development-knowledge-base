## 区别概述

**存储空间上：**
- 结构体是值类型，存储在栈上
- 类是引用类型，存储在堆上

**使用上：**
- 都可以用面向对象的思想去形容一类对象
- 结构体具有封装的特性，但不具备继承和多态的特性
- 结构体不具备继承的特性，所以不能使用 protected 访问修饰符

---

## 细节区别

| 对比项       | 结构体 struct             | 类 class     |
| --------- | ---------------------- | ----------- |
| 存储位置      | 栈（值类型）                 | 堆（引用类型）     |
| 赋值行为      | 拷贝值，互不影响               | 拷贝引用，共享同一对象 |
| protected | ❌ 成员不能使用               | ✅ 可以        |
| 成员初始化     | ❌ 声明时不能指定初始值（无法修改默认值）  | ✅ 可以        |
| 无参构造      | ❌ 不能声明                 | ✅ 可以        |
| 有参构造后无参构造 | ✅ 不会被顶掉                | ❌ 会被顶掉      |
| 析构函数      | ❌ 不能有                  | ✅ 可以        |
| 继承        | ❌ 不能被继承，不能继承           | ✅ 支持        |
| 构造函数要求    | 需要初始化所有成员变量            | 随意          |
| static 修饰 | ❌ 不能被 static 修饰        | ✅ 可以        |
| 同类型成员     | ❌ 不能在自己内部声明和自己一样的结构体变量 | ✅ 可以        |

---

## 结构体的特别之处

> **结构体可以继承接口**，因为接口是行为的抽象。

---

## 如何选择

| 情况                       | 选择                     |
| ------------------------ | ---------------------- |
| 需要继承和多态                  | **class**（直接淘汰 struct） |
| 对象是数据集合（位置、坐标等）          | 优先 **struct**          |
| 经常被赋值传递，改变赋值对象时原对象不想跟着变化 | **struct**（如坐标、向量、旋转）  |
