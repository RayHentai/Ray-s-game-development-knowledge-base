## 构造函数

> 实例化对象时会调用的，用于初始化的函数。不写会默认存在一个无参构造函数。

**规则：**
- 没有返回值
- 函数名和类名必须相同
- 没有特殊需求，一般都是 public
- 可以被重载
- 可以用 `this` 区分同名参数和成员变量
- **有参构造会顶掉默认的无参构造**
- 类中允许自己申明无参构造函数，而**结构体不允许**

```csharp
class Player
{
    public string name;
    public int    hp;

    public Player() { }  // 无参构造

    public Player(string name, int hp)  // 有参构造
    {
        this.name = name;
        this.hp = hp;
    }
}
```

---

### 构造函数特殊写法（this 复用）

> 用 `this` 重用构造函数代码，避免重复逻辑。

```csharp
// 语法：访问修饰符 构造函数名(参数列表) : this(参数1, 参数2, ...)
public Player(string name) : this(name, 100)
{
    // 先调用 this(name, 100)，再执行这里的内容
    // this 后面的值类型、数量对应哪个重载函数就调用哪个
    // this 后可以写常量
}
```

## 析构函数

> 在 Unity 开发中析构函数几乎不使用，了解即可。

**原理：**
- 当垃圾 真正被回收的时候 才会调用函数
- 当引用类型的堆内存被回收时 会调用该函数
- 对于需要手动管理内存的语言，需要在析构函数中做一些内存回收处理，但是C#中存在自动垃圾回收机制GC，所以不怎么用析构函数，除非想在某一个对象被垃圾回收时 做一些特殊处理

**注意：**
- 不写返回值
- 不写修饰符
- 不能有参数
- 函数名和类名相同

```csharp
// 语法：~类名() { }
~Player()
{
    // C# 有自动垃圾回收机制 GC，所以析构函数不常用
    // 规则：不写返回值、不写修饰符、不能有参数
}
```

> 3. 写一个 Ticket 类，有距离变量（构造时赋值），根据距离计算价格（1元/公里），

- 0~100不打折
- 101~200打9.5折
- 201~300打9折
- 300以上打8折
- 有显示方法

```csharp
class Ticket
{
    public uint  distance;
    public float price;

    public Ticket(uint distance)
    {
        this.distance = distance;
        price = GetPrice();
    }

    private float GetPrice()
    {
        if (distance <= 100)       
	        return distance;
        else if (distance <= 200 && distance > 100) 
	        return distance * 0.95f;
        else if (distance <= 300 && distance > 200) 
	        return distance * 0.9f;
        else
	        return distance * 0.8f;
    }

    public void Write()
    {
        Console.WriteLine("{0}公里{1}块钱", distance, price);
    }
}

Ticket t = new Ticket(400);
t.Write();
```
