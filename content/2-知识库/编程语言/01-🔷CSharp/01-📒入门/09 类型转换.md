## 隐式转换（自动）

```csharp
变量名 = 值/变量名;
int i = 100;
long l = i;      // ✅ 小装大，安全
// int j = l;   // ❌ 大装小，报错
```

> 规则：**大范围类型能装小范围类型，反过来不行。**

```
long > int > short > sbyte
ulong > uint > ushort > byte
decimal ❌> double > float

double > float > 所有整型 > char
decimal > 所有整型 > char
string❌ bool❌
```

**特殊规则：**
- 无符号不能装有符号
- 有符号可以装无符号（前提：值在范围内）
- 浮点数可以装任何类型整数（用科学计数法表示）
- `bool` 和 `string` 不参与隐式转换
- `char`无法存储其他类型变量
- `char` 可以隐式转换为 `int`（对应 ASCII 码值）

## 显示转换

### 四种转换方式总览

| 方式         | 场景                | 是否安全       |
| ---------- | ----------------- | ---------- |
| 括号强转       | 数值之间低精度转为高精度      | ⚠️ 可能丢失数据  |
| Parse 法    | 把字符串类型转换成对应的类型    | ⚠️ 格式不对会报错 |
| Convert 法  | 更准确的将各个类型之间进行相互转换 | ✅ 会四舍五入    |
| ToString 法 | 把任意类型转为字符串        | ✅ 安全       |

---

### 括号强转（显式）

```csharp
//语法：(目标类型)变量名
double d = 3.99;
int i = (int)d;   // i = 3，直接截断（不是四舍五入！）
```

**注意：** 
- 可能会出现范围问题，造成异常
- 可能会出现精度问题，造成数值丢失
- 浮点数强转成整型会丢失小数，**不会四舍五入**
- `bool` 和 `string` 不能用括号强转

---

### Parse 法

^2381b2

```csharp
//语法 变量类型.parse("字符串")
int i = int.Parse("123");
float f = float.Parse("3.14");
bool b = bool.Parse("True");   // 必须是 "True" 或 "False"
char c = char.Parse("A");      // 只能是单个字符

//配合异常捕获
try
{
	int i = int.Parse(Console.WriteLine());
}
catch {  }
```

**注意：** 
- 字符串必须能够转换成可以存储的类型，格式不对会**直接崩溃**，建议配合 [[10 异常捕获]] 使用
- `bool`也是可以的 但是一定要是`True`和`False`
- `char`也可以 但是只能是一个字符
- 范围问题导致的异常
- 整型装浮点的精度丢失问题

---

### Convert 法

```csharp
//语法 Convert.To目标类型(变量或常量);
double d = 3.6;
int i = Convert.ToInt32(d);      // i = 4（四舍五入！）

bool flag = true;
int b = Convert.ToInt32(flag);   // true→1，false→0
```

**类型名称对照：** 

| 类型     | 类型名称    |
| ------ | ------- |
| short  | Int16   |
| int    | Int32   |
| long   | Int64   |
| ushort | uint16  |
| uint   | uint32  |
| ulong  | uint64  |
| float  | Single  |
| bool   | Boolean |

**注意：**
- 精度更高，**会四舍五入**
- 可以把`bool`类型转成数值形式
- 用`convert`法转`char`是会转成 `ASKII码`的

---

### ToString 法

>当进行字符串拼接时 会自动调用`tostring`转成`string`

```csharp
//语法 变量.ToString();
int i = 123;
string s = i.ToString();   // "123"
```

---

## 核心区别

| 对比 | 括号强转 | Convert |
|------|---------|---------|
| 浮点→整型 | 直接截断（3.9→3） | 四舍五入（3.9→4） |
| bool→数字 | ❌ 不支持 | ✅ true→1 |

## 练习

> 1. 什么情况下会出现数据类型的隐式转换，请举例说明

- 数据范围不够用时，比如有符号转 float（低精度→高精度）
- 例：`int i = 100; long l = i;` 自动转换

> 2. 请将自己名字的每一个字符转换成数字并打印出来

```csharp
int i1 = '张';
Console.WriteLine(i1);
int i2 = '隽';
Console.WriteLine(i2);
int i3 = '一';
Console.WriteLine(i3);
Console.WriteLine("我的数字编号是" + i1 + i2 + i3);
```

> 3. 显示类型转换有几种方式？他们分别是什么，请举例说明

- 括号强转：低精度强制转高精度，`int i = (int)3.99;`
- Parse 法：字符串转对应类型，`int i = int.Parse("123");`
- Convert 法：精确转换，`int i = Convert.ToInt32(3.6);` → 4
- ToString 法：转 string，`string s = i.ToString();`

> 4. 请将 24069 转成字符，并打印

```csharp
Console.WriteLine(Convert.ToChar(24069));
```

> 5. 提示用户输入姓名、语文、数学、英语成绩，并将输入的3门成绩用整形变量存储

```csharp
Console.WriteLine("请输入姓名"); Console.ReadLine();
Console.WriteLine("请输入语文成绩");
int yuWen  = int.Parse(Console.ReadLine());
Console.WriteLine("请输入数学成绩");
int shuXue = int.Parse(Console.ReadLine());
Console.WriteLine("请输入英语成绩");
int yingYu = int.Parse(Console.ReadLine());
Console.WriteLine("你的成绩是" + yuWen + " " + shuXue + " " + yingYu);
```

## 我的踩坑记录

- Parse 失败会直接崩溃，一定要加 try-catch 或改用 `int.TryParse()`
- 括号强转和 Convert 对小数的处理方式完全不同，容易搞混
