## 理解"写代码"是什么

> 通过写代码命令调用系统或者自定义的 API，让计算机通过这些命令执行行为，以实现特定需求。

**用我自己的话说**：

- 写代码就是给计算机发指令，指令格式必须符合语言规范，计算机才能听懂。

---

## 核心术语

| 术语          | 解释              | 类比理解                    |
| ----------- | --------------- | ----------------------- |
| `IDE`       | 集成开发环境          | 写代码用的软件，如 Visual Studio |
| `API`       | 系统写好的逻辑         | 别人封装好的功能，直接拿来用          |
| `namespace` | 命名空间            | 工具包，把相关的类组织在一起          |
| `class`     | 类               | 工具本身                    |
| `Main`      | 主函数             | 工具能做的事，程序从这里开始执行        |
| `using`     | 引用命名空间          | 告诉程序要用哪个工具包             |
| 语句块         | 被大括号 `{}` 包裹的部分 | 代码的"范围"，影响变量生命周期        |

---

## C# 程序基本结构

```csharp
using System;               // 引用命名空间（工具包）

namespace MyProject         // 我自己的命名空间（工具包名）
{
    class Program           // 类（工具）
    {
        static void Main()  // 主函数，程序入口（工具能做的事）
        {
            // 代码写在这里
        }
    }
}
```

---

## 注意

- **语句块会影响变量的生命周期**：在 `{}` 内声明的变量，出了 `{}` 就不再存在
- 每条语句末尾加分号 `;`
