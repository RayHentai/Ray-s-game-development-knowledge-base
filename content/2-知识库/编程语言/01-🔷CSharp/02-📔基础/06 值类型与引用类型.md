## 核心理解

> 值类型赋值 = 复印一份，改副本不影响原件。
> 引用类型赋值 = 两人共用一张地图，一人改了另一人也看到变化。

---

## 分类

| 类型       | 包含                                     |
| -------- | -------------------------------------- |
| **值类型**  | 结构体、14种基础变量类型（int/float/bool/char等）、枚举 |
| **引用类型** | 数组、string、类                            |

---

## 使用上与存储上区别

---

### 使用上的区别

```csharp
// 值类型：互相赋值后独立，改一个不影响另一个
int a = 10;
int b = a;
b = 20;
Console.WriteLine(a);  // 10，a 没有变

// 引用类型：赋值后指向同一块内存，改一个另一个也变
int[] arr1 = { 1, 2, 3 };
int[] arr2 = arr1;
arr2[0] = 99;
Console.WriteLine(arr1[0]);  // 99，arr1 也变了
```

---

### 存储区域不同（根本原因）

>值类型和引用类型存储在的内存区域是不同的，存储方式是不同的，所以造就了它们在使用上的区别

|      | 值类型           | 引用类型        |
| ---- | ------------- | ----------- |
| 存储位置 | **栈（Stack）**  | **堆（Heap）** |
| 特点   | 系统分配，自动回收，小而快 | 手动申请/释放，大而慢 |
| 赋值行为 | 拷贝值本身         | 拷贝内存地址（引用）  |

>如果New了新数组 就是在堆中开了一个新房间

---

## 特殊：string 是引用类型但表现像值类型

```csharp
string s1 = "hello";
string s2 = s1;
s2 = "world";
Console.WriteLine(s1);  // "hello"，s1 没有变
```

> C# 中 `string` 每次修改都会创建新的字符串对象，所以表现出"它变我不变"。

>代价是：频繁修改 string 会产生大量内存垃圾，应使用 `StringBuilder` 优化。[[26 StringBuilder]]

## 练习

> 1. 请问打印结果为多少？

```csharp
int a = 10;
int b = a;
Console.WriteLine(a);
// 答：10（值类型赋值是拷贝，b 的变化不影响 a）
```

> 2. 请问打印结果为多少？

```csharp
int[] a = new int[] { 10 };
int[] b = a;
b[0] = 20;
Console.WriteLine(a[0]);
// 答：20（引用类型赋值，a 和 b 指向同一数组）
```

> 3. 请问打印结果是多少？

```csharp
string str  = "123";
string str2 = str;
str2        = "321";
Console.WriteLine(str);
// 答：123（string 表现出值类型特征，str2 重新赋值不影响 str）
```

> 4. 请问打印结果是多少？

```csharp
int[] a = new int[] { 10 };
int[] b = a;
b = new int[5];
Console.WriteLine(a[0]);
// 答：10（b = new int[5] 让 b 指向新数组，不影响 a 原来指向的数组）
```
