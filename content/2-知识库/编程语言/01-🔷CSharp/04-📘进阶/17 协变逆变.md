## 核心理解

**协变（out）：** 和谐的变化，自然的变化。因为里氏替换原则父类可以装子类，所以子类变父类（string → object）感受是和谐的。父类泛型委托装子类泛型委托。

**逆变（in）：** 逆常规的变化。父类变子类（object → string）感受是不和谐的。子类泛型委托装父类泛型委托。

---

## 代码示例

**规则**
- **只有泛型接口和泛型委托可以使用协变逆变**
- `out` 修饰的泛型：只能作为**返回值**（协变）
- `in` 修饰的泛型：只能作为**参数**（逆变）
- 用 out/in 修饰的泛型委托，可以互相装载（有父子关系的泛型）

**作用：**
- 用来修饰泛型替代符的 只能用在泛型委托和泛型接口中

```csharp
// 协变 out：泛型只能作为返回值
delegate T TestOut<out T>();

// 逆变 in：泛型只能作为参数
delegate void TestIn<in T>(T t);

class Father { }
class Son : Father { }

// 协变：父类容器可以装子类委托（因为子类返回值可以当父类用）
TestOut<Son> s = () => new Son();
TestOut<Father> f = s; // ✅ 父类泛型委托装子类泛型委托
Father fa = f();

// 逆变：子类容器可以装父类委托
TestIn<Father> f1 = (a) => { };
TestIn<Son> s1 = f1; // ✅ 子类泛型委托装父类泛型委托
s1(new Son());
```

## 练习

> 1. 请描述协变逆变有什么作用


- 用来修饰泛型替代符，只能用在泛型委托和泛型接口中。
- 用 `out` 修饰的泛型只能作为返回值（协变），用 `in` 修饰的泛型只能作为参数（逆变）。
- 用来遵循里氏替换原则，使用 `out`/ `in` 修饰的泛型委托可以互相装载（有父子关系的泛型）。
