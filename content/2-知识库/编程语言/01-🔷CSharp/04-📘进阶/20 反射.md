## 相关概念

- **程序集：** 经由编译器编译得到的，供进一步编译执行的那个中间产物。在Windows系统中一般表现为.dll(库文件) 或者是.exe(可执行文件)的格式。简单说 程序集就是我们写的一个代码集合，我们现在写的所有代码，最终都会被编译器翻译为一个程序集供别人使用。

- **元数据：** 描述数据的数据，程序中的类、函数、变量等信息就是元数据，保存在程序集中

- **类库：** 就是一个 dll 文件，方便在其他程序集中调用

- **反射：** 程序正在运行时，查看自身或其他程序集的元数据，并实例化对象、调用方法。一个运行的程序查看本身或者其他程序的元数据的行为就叫做反射。简单说，在程序运行时 通过反射，可以得到其他程序集或自己程序集代码的各种信息，类、函数、变量、对象等等实例化它们、执行它们、操作它们。

---

## 核心理解

> 一个运行的程序查看本身或者其他程序的元数据的行为就叫做反射。
> 作用：提高程序的拓展性和灵活性，程序运行时得到所有元数据，实例化对象，操作对象。

**作用：**
- 因为反射可以在程序编译后获得信息 所以它提供好了程序的拓展性和灵活性
- 程序运行时得到所有元数据 包括元数据的特性
- 程序运行时 实例化对象 操作对象
- 程序运行时创建新对象 用这些对象执行任务

## Type（反射的基础）

**Type：**
- 类的信息类
- 是反射功能的基础
- 是访问元数据的主要方式
- 使用Type 的成员获取有关类型声明的信息
- 有关类型的成员（构造函数、方法、字段、属性、类的事件）

```csharp
// 获取 Type 的三种方式（以下三个 type 指向的堆内存都是一样的）
Type t1 = 对象.GetType(); // 通过 object 的 GetType() 方法
Type t2 = typeof(类名); // 通过 typeof 传入类名
Type t3 = Type.GetType("命名空间.类名"); // 通过类名字符串（必须包含命名空间）

// 得到程序集信息
Assembly assembly = t1.Assembly;
```

## 反射的使用

### Info相关（字段信息）

```csharp
// 需要引用命名空间 System.Reflection
using System.Reflection;
//元数据
class Test
{
	private int i = 1；
	public int j = 0;
	public string str = "123";
	
	public Test() { }
	public Test(int i) { this.i = i; }
	public Test(int i, string str) :this(i) { this.i = i; }
	public void Speak() { Console.WriteLine(i) }
}
//得到Type
Type t = typeof(Test);

// 获取所有公共成员 GetMembers
MemberInfo[] members = t.GetMembers(); 

// 获取构造函数 GetConstructors
ConstructorInfo[] ctors = t.GetConstructors();

// 无参构造
ConstructorInfo info = t.GetConstructor(new Type[0]); //获取
Test obj = info.Invoke(null) as Test// 执行无参构造 没有参数传null
Console.WhiteLine(obj.j); //0

// 有参构造
ConstructorInfo info2 = t.GetConstructor(new Type[] { typeof(int) });//获取
Test obj2 = info2.Invoke(new object[] {2}) as Test;//执行有参构造
Console.WhiteLine(obj2.i); // 2

//多个参数的有参构造
ConstructorInfo info3 = t.GetConstructor(new Type[] { typeof(int), typeof(string) });//获取
Test obj3 = info3.Invoke(new object[] {4, "444444"}) as Test;//执行有参构造 传入参数
Console.WhiteLine(obj3.str); // "444444"

// 获取公共类的成员变量 GetFields
FieldInfo[] fields = t.GetFields();//得到所有成员变量字段
FieldInfo infoJ = t.GetField("j");//得到指定名称的成员变量字段
Console.WriteLine(infoJ); // Int32 j

//获取和设置对象的值 GetValue SetValue
FieldInfo infoJ = t.GetField("j");//得到指定名称的成员变量字段
Test test = new Test();
test.j = 99;
int value = infoJ.GetValue(test);//获取
Console.WriteLine(value); // 99
value = infoJ.SetValue(test, 100);//设置
Console.WriteLine(value); // 100

// 获取成员方法 GetMethods 以 String 的 Substring 为例 
Type strType = typeof(string);
MethodInfo[] methods = strType.GetMethods();//得到所有成员方法字段
MethodInfo subStr = strType.GetMethod("Substring", new type[] { typeof(int), typeof(int) });//得到 名为Substring 参数为 int 和 int 的方法字段
//方法的不同重载可以用Type数组指定传入参数类型
string str = "Hello,World!";
object result = subStr.Invoke(str, new object[] { 7, 5 });//为 str 执行 subStr字段对应的方法 第一个参数为 7 第二个参数为 5
//如果是静态方法 第一个参数填null就行
Console.WriteLine(result);//"orld!"

//获取其他数据类型字段的方法
//枚举 GetEnumName GetEnumNames
//事件 GetEvent GenEvents
//接口 GetInterface GetInterfaces
//属性 GetProperty GetPropertys
//等等......
```

### Assembly（加载其他程序集） 

- 主要用来加载其他程序集
- 加载后 才能用Type来使用其他程序集中的信息
- 如果想要使用不是来自程序集中的内容 需要先加载程序集
- 比如 dll文件（库文件）
- 简单的把库文件看成一种代码仓库 它提供给使用者一些可以直接拿来用的变量、函数或类

```csharp
//加载
//一般用来加载在同一文件下的其他程序集
Assembly asembly = Assembly.Load("程序集名称")
//一般用来加载不在同一文件下的其他程序集
Assembly asembly = Assembly.LoadFrom("包含程序集清单的文件的名称或路径");
Assembly asembly = Assembly.LoadFile("要加载的文件的完全限定路径");

//实现
//先加载一个指定程序集
Assembly asembly = Assembly.LoadFrom("包含程序集清单的文件的名称或路径");
Type[] = types = asembly.GetType();//得到程序所有类型
//加载一个类对象
Type type = asembly.GetType("上面得到的类名之一");//得到这个类对象的type

//进行反射
//GetFiele 得到成员变量字段
//Activartor.CreateInstance 实例化对象
//GetMethod 得到成员方法字段
//Invoke 调用
```

## 练习

> 1. 新建一个类库工程，有一个 Player 类，有姓名/血量/攻击力/防御力/位置等信息，有一个无参构造函数；再新建一个控制台工程，通过反射的形式使用类库工程生成的 dll 程序集实例化一个 Player 对象

```csharp
// 加载程序集
Assembly assembly = Assembly.LoadFrom(@"类库测试路径\类库测试.dll");
Type[] type = assembly.GetTypes();
for (int i = 0; i < type.Length; i++) 
	Console.WriteLine(type[i]);

Console.WriteLine("*********************");

// 加载类信息
Type PlayerType = assembly.GetType("类库测试.Player");
MemberInfo[] members = PlayerType.GetMembers();
for (int i = 0; i < members.Length; i++) 
	Console.WriteLine(members[i]);

Console.WriteLine("*********************");

// 实例化对象
object playerObj = Activator.CreateInstance(PlayerType);
Console.WriteLine(playerObj);
```
