## 核心理解

> 特性（Attribute）= 允许向程序的程序集添加元数据的语言结构。

- 用于保存程序结构信息的某种特殊类型的类
- 特性提供功能强大的方法将声明信息与C#代码（类型、方法、属性等）相关联特性与程序实体关联后 即可在运行时使用反射查询特性信息
- 特性的目的时告诉编译器把程序结构的某组元素嵌入程序集中，它可以放置在几乎所有的声明类中（类、变量、函数等等声明）

>简单说 特性本质是个类

- 我们可以利用特性为元数据添加额外信息 比如一个类、成员变量、成员方法等，为它们添加更多额外信息，之后可以通过反射来获取这些额外信息

## 自定义特性 与使用 自定义特性

- 使用时本质上就是在调用特性类的构造函数 
- 相当远生成了一个类对象 关联另外一个类 
- Type得到类对象的时候 可以得到特性类的关联信息 
- 在 类、函数、变量上一行 都可以写，表示它们具有该特性信息

```csharp
// 继承特性基类，命名规范：名字后面加 Attribute
class MyCustomAttribute : Attribute
{
    public string info;
    public MyCustomAttribute(string info) 
    { 
	    this.info = info; 
    }
    public
}

// 在类/变量/函数上面使用（本质是调用特性类的构造函数）括号内填构造函数的参数
//调用时 Attribute 默认被省略
[MyCustom("这是一个测试类")]
class Test
{
	[MyCustom("这是一个成员变量")]
	public string name;
	[MyCustom("这是一个用于测试的方法")]
	public void Test1([MyCustom("这是函数参数")]int a) { }
}

```

## 通过反射获取特性

```csharp
//元数据
class MyCustomAttribute : Attribute
{
    public string info;
    public MyCustomAttribute(string info) 
    { 
	    this.info = info; 
    }
    public TestFun() { }
}

[MyCustom("这是一个测试类")]
class Test
{
	[MyCustom("这是一个成员变量")]
	public string name;
	[MyCustom("这是一个用于测试的方法")]
	public void Test1([MyCustom("这是函数参数")]int a) { }
}
Test test = new test();
Type t = test.GetType();
// 判断是否使用了某个特性
//参数1 特性的类型
//参数2 代表是否搜索继承链（属性和事件忽略此参数）
if (t.IsDefined(typeof(MyCustomAttribute), false))
    Console.WriteLine("使用了 MyCustom 特性");

// 获取所有特性
object[] attrs = t.GetCustomAttributes(true);  // true = 搜索继承链
//找到类信息里面的特性类信息 ​转化为特性类 就可以直接调用成员变量或者函数
for (int i = 0; i < attrs.Length; i++)
{
	if (array[i] is MyCustomAttribute)
	{
		string str = (array[i] is MyCustomAttribute).info; // 得到特性类的成员信息
		Console.WriteLine(str);
		(array[i] is MyCustomAttribute).TestFun();//调用特性的方法
	}
}

// 获取字段的特性
FieldInfo field = t.GetField("name");
var fieldAttr = field.GetCustomAttribute(typeof(MyCustomAttribute));
if (fieldAttr != null) Console.WriteLine("name 字段有特性");
```

## 限制特性的使用范围

```csharp
// AttributeUsage 写在特性前面
[AttributeUsage(AttributeTargets.Class | AttributeTargets.Struct, // 能用在哪些地方
AllowMultiple = true, // 是否允许多个特性实例用在同一个目标上
Inherited = true // 特性是否能被派生类和重写成员继承
)]
class MyCustomAttribute : Attribute { }
```

## 系统自带特性

```csharp
// 过时特性 Obsolete（true = 直接报错，false = 警告）
//用于提示用户 使用的方法等成员以及过时 建议使用新方法 一般加在函数前
[Obsolete("请使用新方法 NewMethod", false)] //显示内容 提示类型
public void OldMethod() { }

// 调用者信息特性（需要 using System.Runtime.CompilerServices）
void Log(
[CallerFilePath] string file = "", //哪个文件调用
[CallerLineNumber] int line = 0, //哪一行调用
[CallerMemberName] string member = "" //哪个函数调用
)
{
    Console.WriteLine($"调用于 {file} 第 {line} 行的 {member}");
}

// 条件编译特性（需要 using System.Diagnostics）
//和 #define配合使用 主要用在一些调试代码上 有时想执行有时不想执行的代码
[Conditional("DEBUG")]//参数为 define标识
void DebugLog(string msg) 
{ 
	Console.WriteLine(msg); 
}

// 外部 DLL 函数特性（需要 using System.Runtime.InteropServices）
//用来标记非.Net(C#)的函数 表明该函数在一个外部的Dll中定义 一般用来调用 C 或 C++的DLL包写好的方法
[DllImport("user32.dll")]
static extern int MessageBox(IntPtr hWnd, string text, string caption, int type);
```

## 练习

> 1. 为反射练习题中的 Player 对象中随便为一个成员变量加一个自定义特性；实现反射练习题的要求，但当在设置加了自定义特性的成员变量时，在控制台中打印"非法操作，随意修改XXX成员"

```csharp
// 类库工程中：
class MyCustomAttribute : Attribute { }

public class Player
{
    [MyCustom] // 加了自定义特性
    public string name;
    public int hp, atk, def;
    public Player() { }
}

// 控制台工程中：
Assembly assembly = Assembly.LoadFrom(@"类库测试路径\类库测试.dll");
Type PlayerType = assembly.GetType("类库测试.Player");
object playerObj = Activator.CreateInstance(PlayerType);

FieldInfo[] fields = PlayerType.GetFields();
for (int i = 0; i < fields.Length; i++) 
	Console.WriteLine(fields[i]);

Console.WriteLine("*********************");

// 得到自定义特性的 Type
Type attribute = assembly.GetType("类库测试.MyCustomAttribute");
FieldInfo fieldStr = PlayerType.GetField("name");

// 判断有没有特性
if (fieldStr.GetCustomAttribute(attribute) != null)
    Console.WriteLine("非法操作，随意修改name成员");
else
    fieldStr.SetValue(playerObj, "Test_改变值");
```
