## 核心理解

> 事件是基于委托的存在，是一种特殊的变量类型，是委托的安全包裹，让委托的使用更具有安全性。

---

## 语法

**规则：**
- 作为成员变量存在于类中
- 事件和委托的使用一模一样，只是有细微差异 

**注意：**
- 不能 在类外部 赋值 会报错
- 不能 在类外部 调用 会报错
- 可以 加减、添加、移除记录的函数
- 只能在类中封装调用
- 只能作为成员存在于类、接口与结构体中
- 不能 作为临时变量在 函数中使用

```csharp

class Test
{
	// 访问修饰符 event 委托类型 事件名
	public event Action<int> testEvent;
	public void Fun() { }
	public void Test1()
	{
		testEvent = Fun; 
		testEvent.Invoke();
		testEvent();
	}
}
Test t = new test();
t.testEvent = Fun;// ❌报错 
t.testEvent.Invoke();// ❌报错 
t.testEvent();// ❌报错 
t.testEvent += Fun;// += / -= 是可以的
event Action a = fun;//报错❌
Action a = fun;//委托可以作为临时变量

```

## 事件的限制（与委托的区别）

|        | 委托   | 事件             |
| ------ | ---- | -------------- |
| 类外部赋值  | ✅ 可以 | ❌ 不能（只能 +=/-=） |
| 类外部调用  | ✅ 可以 | ❌ 不能           |
| 作为临时变量 | ✅ 可以 | ❌ 不能           |
| 作为成员   | ✅ 可以 | ✅ 可以           |

> 事件可以加减（`+=/-=`），但不能在类外部赋值或调用，只能在类中封装调用。

---

## 作用

- 防止外部随意置空委托
- 防止外部随意调用委托
- 事件相当于对委托进行了一次封装，让其更加安全

## 练习

> 1. 有一个热水器，包含一个加热器，一个报警器，一个显示器我门给热水器通上电，当水温超过95度时

1. 报警器会开始发出语音，告诉你水的温度
2. 显示器也会改变水温提示，提示水已经烧开了


```csharp
class JiaReQi
{
    public int value = 0;
    public event Action<int> hot;

    public void AddHot()
    {
        int index = 0;
        while (true)
        {
            if (value == 100) break;
            if (index % 9999999 == 0)
            {
                ++value;
                if (value >= 95)
                {
                    if (hot != null) hot(value);
                    hot = null;
                }
                Console.WriteLine("加热到{0}°", value);
                index = 0;
            }
            ++index;
        }
    }
}

class XianShiQi { public void ShowInfo(int i) { Console.WriteLine("语音提示：当前水温{0}°", i); } }
class BaoJingQi { public void ShowInfo(int i) { Console.WriteLine("当前水温{0}°,水烧开了！", i); } }

BaoJingQi b = new BaoJingQi(); XianShiQi x = new XianShiQi();
JiaReQi j = new JiaReQi();
j.hot += b.ShowInfo;
j.hot += x.ShowInfo;
j.AddHot();
```
