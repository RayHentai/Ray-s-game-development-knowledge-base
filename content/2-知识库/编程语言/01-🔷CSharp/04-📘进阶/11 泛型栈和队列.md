## 核心理解

> 泛型版的 `Stack<T>` 和 `Queue<T>`，用法与非泛型版本完全相同，但类型更安全、性能更好。

```csharp
using System.Collections.Generic;
Stack<int> stack = new Stack<int>();
Queue<string> queue = new Queue<string>();
// 使用上同 Stack、Queue
```

## 练习

> 1. 自己总结一下：数组、List、Dictionary、Stack、Queue、LinkedList 这些存储容器应该如何选择

```
普通线性表：
  - 数组：固定的不变的一组数据
  - List：经常改变，经常通过下标查找
  - LinkedList：不确定长度，经常临时插入改变，查找不多

先进后出 → Stack
  - 适合利用先进后出存储特点的逻辑，如 UI 面板显隐规则

先进先出 → Queue
  - 适合利用先进先出存储特点的逻辑，如消息队列：有了就往里放，然后慢慢依次处理

键值对 → Dictionary
  - 需要频繁查找的，有对应关系的数据
  - 如：道具ID → 道具信息，怪物ID → 怪物对象
```
