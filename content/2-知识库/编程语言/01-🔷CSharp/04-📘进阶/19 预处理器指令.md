# ⚙️ 19 预处理器指令

**所属模块**：[[00 CSharp 进阶阶段总览]]
**查阅次数**：0

---

## 核心理解

> 预处理器指令让代码在还没有编译之前就进行一些预处理判断。
> 在 Unity 中用来进行平台或版本的判断，决定不同平台/版本使用不同的代码逻辑。

---

## 相关概念

- **编译器：** 将源语言程序（C#）翻译为目标语言程序（二进制）的翻译程序
- **源语言程序：** 某种程序设计语言写成的，比如C#、C、C++、Java等语言写的程序
- **目标语言程序：** 二进制数表示的伪机器代码写的程序
- **预处理器指令：** 指导编译器在实际编译开始之前对信息进行预处理，以 `#` 开始，**不是语句，所以不以分号结束**，目前常用的折叠代码块 `#region / #endregion` 就是预处理器指令

---

## 常用指令

```csharp
// 第一组：定义/取消符号（写在脚本文件最前面）
#define 符号名 // 定义一个符号（类似没有值的变量）
#undef 符号名 // 取消 define 定义的符号，让其失效

// 第二组：条件编译（和 if 语句规则一样，结尾要加 endif）
//如果发现这个符号 就会被编译器翻译
#if 符号名
#elif 符号名 // 就是 else if
#else
#endif // 与 if 不同，结尾要加 endif

// 可以通过逻辑与/或进行多种符号的组合判断
#if UNITY_EDITOR && DEBUG
    // 只在编辑器且调试模式下编译
#endif

// 第三组：编译警告/错误
#warning 消息 // 告诉编译器报警告（false 效果）
#error 消息 // 告诉编译器报错误（true 效果，中止编译）
// 一般配合 #if 使用
```

---

## 代码示例

```csharp
#define DEBUG

#if DEBUG
    Console.WriteLine("调试模式");
#elif RELEASE
    Console.WriteLine("发布模式");
#else
    Console.WriteLine("未知模式");
#endif
```

## 练习

> 1. 请说出至少4种预处理器指令


- define   — 定义符号
- undef    — 取消定义符号
- if — 条件编译开始
- elif — 条件编译分支（else if）
- else — 条件编译默认分支
- endif — 条件编译结束
- warning — 生成编译警告
- error — 生成编译错误（中止编译）


> 2. 请使用预处理器指令实现：写一个函数计算两个数，当是 Unity5 版本时算加法，当是 Unity2017 版本时算乘法，当是 Unity2020 版本时算减法，都不是返回0

```csharp
public int Add(int a, int b)
{
#if Unity5
    return a + b;
#elif Unity2017
    return a * b;
#elif Unity2020
    return a - b;
#else
    return 0;
#endif
}
```
