## var 隐式类型

> 是一种特殊的变量类型，可以用来表示任意类型的变量。

**注意：**
- var 不能作为类的成员，只能用于临时变量（函数语句块中）
- var 必须初始化
- var 初始化后无法变更类型

**作用：**
- 偷懒
- 不确定类型时

```csharp
var i = 10; // 推断为 int
var str = "Hello"; // 推断为 string
var list = new List<int>();
```

## 可空类型 ?

^6a3bc8

- ?本质上是个语法糖

```csharp
// 值类型不能赋值为空，加 ? 后可以赋值为空
int? nullableInt = null;
bool? nullableBool = null;

// 判断是否为空
if (nullableInt == null) { }
if (nullableInt.HasValue) { }

// 安全获取值（如果为空，返回值类型默认值）()中的参数填的就是返回的默认值
int val = nullableInt.GetValueOrDefault(); // 返回 0
int val2 = nullableInt.GetValueOrDefault(-1); // 指定默认值 -1
// 注意：GetValueOrDefault 不会给变量赋值

// 引用类型也可以用 ?（主要用于可空警告）
object obj = null;
obj?.Tostring();// 如果obj为空不会执行
```

## 空合并操作符 ??

```csharp
// 如果左边值为 null 就返回右边值，否则返回左边值
// 只要是可以为 null 的类型都能用
int? a = null;
int b = a ?? -1;  // b = -1（因为 a 为 null）

string str = null;
string result = str ?? "默认值";  // result = "默认值"
```

## 值类型和引用类型补充

---

### 如何判断值类型和引用类型
- 按 F12 进到类型内部，是 `class` 就是引用，是 `struct` 就是值
- 结构体中的值类型：存储在栈中；结构体中的引用类型：堆中存储具体内容，栈上只存地址
- 类中的值和引用：无论是值类型还是引用类型，都存储在堆中（类本身是引用类型，在堆上）

---

### 语句块

```
结构
	命名空间
    ↓
    类、接口、结构体
    ↓
    函数、属性、索引器、运算符重载（类、接口、结构体）
    ↓
    条件分支、循环

分类
    上层语句块
    ｜ 类、结构体
    中层语句块
    ｜ 函数
    底层语句块
    ｜ 条件分支、循环

书写位置
    逻辑代码写在 函数、条件分支、循环 中底层语句块中
    变量声明在 上、中、底层
    成员变量声明在上层
    临时变量声明在中、底层
```

---

### 变量的生命周期

>**编程时 大部分都是临时变量，在中底层声明的临时变量（函数、条件分支、循环语句块等）**
>**语句块执行结束时，大括号中的内容，没有被记录的对象将会被回收或变成垃圾**

**值类型：** 被系统自动回收（弹栈）

**引用类型：** 栈上用于存储地址的房间被系统自动回收，堆中具体的内容变成垃圾，待下次GC回收，**堆中的垃圾此时并未被回收**

**改变声明周期**
- 想要变量不被回收或者不变垃圾 必须将其记录下来 
- 在更高层级记录 或者使用静态全局记录变量

---

### 结构体中的值和引用

- 结构体本身是值类型
- 前提 该结构体没有作为其他类的成员
- 在结构体中的值 栈中存储值具体的内容
- 在结构体中的引用 堆中存储引用具体内容
- 引用类型始终存储在堆中
- 真正通过结构体使用其中引用类型时只是顺藤摸瓜
- 结构体中的引用类型 栈上只会存它的地址

---

### 类中的值和引用

- 类本身是引用类型
- 在类中的值 堆中存储具体的值
- 在类中的引用 堆中存储具体的值
- 值类型直接存在房间 引用一定有一个地址指向一个房间 不管在哪都是这样

---

### 数组中的值和引用

- 数组本身是引用类型
- 值类型数组 堆中房间存具体内容
- 引用类型数组 堆中房间存地址

---

### 结构体继承接口

>利用里氏替换原则 接口容器装结构体存在装箱拆箱

```csharp
interface Itest 
{ 
	int Value
	{
		get;
		set;
	}
}
public Struct TestStruct : ITest
{
	public int value;
	int Value
	{
		get
		{
			return value;
		}
		set
		{
			this.value = value;
		}
	}
}
TestStruct obj1 = new TestStruct();
obj1.value = 1;
TestStruct obj2 = obj1;
obj2.value = 2;
Console.WriteLine(obj1.value) //1
Console.WriteLine(obj2.value) //2
ITest iObj1 = obj1; //装箱
ITest iObj2 = obj1;
iObj2.Value = 99;
Console.WriteLine(obj1.value) //99
Console.WriteLine(obj2.value) //99
TestStruct obj3 = (TestStruct)iObj1;//拆箱
```
