## 核心理解

链表是一种**通过指针串联在一起的线性结构**，每一个节点由两部分组成，
- 一个是数据域
- 一个是指针域（存放指向下一个节点的指针）

最后一个节点的指针域指向null（空指针的意思）。
链表的入口节点称为链表的头结点也就是head。
![[链表 图示.png|364]]

```csharp title:定义
public class Node<T>
{
    // 节点存储的数据
    public T Data { get; set; }
    // 指向下一个节点的引用
    public Node<T> Next { get; set; }
    // 节点的构造函数，用于初始化节点
    public Node(T data)
    {
        Data = data;
        Next = null; // 初始时没有下一个节点，因此设为 null
    }
}
```

**1. 单链表**
单链表中的指针域只能指向节点的下一个节点

## 关键特性

**存储方式：在内存中不是连续分布的**

链表是通过指针域的指针链接在内存中各个节点。
所以链表中的节点在内存中不是连续分布的 ，而是**散乱分布在内存中的某地址上**，分配机制取决于操作系统的内存管理。

### 虚拟头结点

**作用：** 统一处理头节点和普通节点的逻辑，避免对头节点单独做判断

**原理：** 在链表头部新建一个不存储有效数据的虚拟节点，让原来的头节点也有了前驱节点，这样删除、插入等操作对所有节点包括原头节点都能用同一套逻辑处理。最后返回 `dummy.next` 而不是原来的 `head`，因为原头节点可能已经被删除了。

**什么时候用：** 需要对链表头节点做增删操作时，用虚拟头节点可以简化代码。

声明一个虚拟头节点 指向当前链表的头节点 `ListNode dummy = new ListNode(0, head)`
声明一个指针节点 指向虚拟头节点  `ListNode cur = dummy`

遍历指针 ``while (cur.next != null)``

进行判断 
如果指针下一个节点的值等于给定值 `if (cur.next.val == val)`  
指针的下一个节点指向指针的下下个节点 `cur.next = cur.next.next`

否则（不等于给定值）移动指针 `cur = cur.next`

出循环 返回虚拟头节点的下一个节点，即头节点 `return dummy.next`

```
dummy = new ListNode(0, head)
cur = dummy
while (cur.next != null)
	if (cur.next.val == val)
		cur.next = cur.next.next
	else
		cur = cur.next
return dummy.next
```
