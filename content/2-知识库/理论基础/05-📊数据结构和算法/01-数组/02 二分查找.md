## 题目描述

>给定一个 `n` 个元素有序的（升序）整型数组 `nums` 和一个目标值 `target`  ，写一个函数搜索 `nums` 中的 `target`，如果 `target` 存在返回下标，否则返回 `-1`。
>你必须编写一个具有 `O(log n)` 时间复杂度的算法。

  
**示例 1:**

**输入:** `nums` = \[-1,0,3,5,9,12], `target` = 9
**输出:** 4
**解释:** 9 出现在 `nums` 中并且下标为 4

**示例 2:**

**输入:** `nums` = \[-1,0,3,5,9,12], `target` = 2
**输出:** -1
**解释:** 2 不存在 `nums` 中因此返回 -1

**提示：**
1. 你可以假设 `nums` 中的所有元素是不重复的。
2. `n` 将在 `[1, 10000]`之间。
3. `nums` 的每个元素都将在 `[-9999, 9999]`之间。

## 核心思路

**这道题目的前提是数组为有序数组**，同时题目还强调**数组中无重复元素**，因为一旦有重复元素，使用二分查找法返回的元素下标可能不是唯一的，这些都是使用二分法的前提条件，当大家看到题目描述满足如上条件的时候，可要想一想是不是可以用二分法了。

二分查找涉及的很多的边界条件，逻辑比较简单，但就是写不好。例如到底是 `while(left < right)` 还是 `while(left <= right)`，到底是`right = middle`呢，还是要`right = middle - 1`呢？

大家写二分法经常写乱，主要是因为**对区间的定义没有想清楚，区间的定义就是不变量**。要在二分查找的过程中，保持不变量，就是在while寻找中每一次边界的处理都要坚持根据区间的定义来操作，这就是**循环不变量**规则。

写二分法，区间的定义一般为两种，左闭右闭即\[left, right]，或者左闭右开即\[left, right)。

下面我用这两种区间的定义分别讲解两种不同的二分写法。

## 解法二：左闭右闭（最优）

**核心思路：**

**时间复杂度：** O(log n)
**空间复杂度：** O(1)

```csharp
public class Solution {
    public int Search(int[] nums, int target) {
        int left = 0;
        int right = nums.Length - 1;
        while (left <= right){
            int middle = left + (right - left) / 2;
            if (nums[middle] > target)
                right = middle - 1;
            else if (nums[middle] < target)
                left = middle + 1;
            else
                return middle;
        }
        return -1;
    }
}
```

## 重要知识点

**溢出问题**

计算中间值用 left + (right - left) / 2 而不是 (left + right) / 2 
**原因：** 两个大整数相加可能超过 int 最大值导致溢出

举个极端的例子就清楚了。

假设 left = 2,000,000,000，right = 2,000,000,000，int 的最大值是 2,147,483,647。

```
(left + right) / 2
= 2,000,000,000 + 2,000,000,000
= 4,000,000,000   ← 超过 int 最大值，溢出，变成负数，结果错误

left + (right - left) / 2
= 2,000,000,000 + (2,000,000,000 - 2,000,000,000) / 2
= 2,000,000,000 + 0
= 2,000,000,000   ← 正确
```

本质是把"先加再除"改成"先算差再除再加"，中间过程的数值始终不会超过 right 的大小，就不会溢出。

**循环不变量**

因为在整个循环执行过程中，有一个性质始终保持不变。
在二分查找里这个不变的性质是：**target 如果存在，一定在 [left, right] 这个区间内。**
每次循环不管怎么缩小区间，这句话始终成立：
- `nums[middle] > target` → target 在左边，right 缩小，但 target 仍在 [left, right] 内
- `nums[middle] < target` → target 在右边，left 扩大，但 target 仍在 [left, right] 内
- 循环结束还没找到 → 区间为空，target 不存在

这个"target 始终在 [left, right] 内"的性质就是循环不变量，它在循环的每一次迭代前后都成立。

所以循环不变量的实际意义是：**在写边界条件之前，先想清楚你的区间定义是什么，然后所有边界处理都围绕这个定义来写，保证那个性质在每次迭代后依然成立。** 这样代码就不会出现莫名其妙的差一错误（off-by-one error）。

`while (left <= right)` 的含义是：**只要区间里还有元素，就继续搜索。**

当 left == right 时，区间里还剩一个元素 `[left, left]`，这个元素还没被检查过，所以必须进循环。用 `<=` 才能覆盖这种情况。

如果写成 `while (left < right)`，当 left == right 时就退出了，最后那个元素永远不会被检查到，遇到 target 正好是最后剩下的那个元素时就会漏掉，返回 -1。
