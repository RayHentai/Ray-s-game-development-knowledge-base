用最直白的方式解释：

**时间复杂度**

衡量代码运行速度随数据量增长的趋势，用 `O()` 表示。

```
一层for循环遍历n个元素    → O(n)
两层for循环嵌套           → O(n²)
每次把问题规模砍一半（二分）→ O(log n)
不管数据量多少都是固定操作 → O(1)
```

比如暴力解法两层循环是 `O(n²)`，滑动窗口一层循环是 `O(n)`，数据量大的时候差距非常明显。

**注意：判断时间复杂度看的是总共执行了多少次操作，不是循环嵌套层数**
例如：一个while里套四个for，只要总操作数是`n²`，时间复杂度就是`O(n²)`

**空间复杂度**

衡量代码额外占用内存随数据量增长的趋势。

```
只声明了几个变量（left、right、sum）→ O(1)  额外空间固定
新建了一个和输入等长的数组          → O(n)  额外空间随输入增长
```

比如有序数组平方，你新建了 `newArray`，空间复杂度就是 `O(n)`。

**约定俗成的规则：** 
- 输入参数不计入空间复杂度 
- 返回值不计入空间复杂度 
- 只计算方法内部为辅助计算额外申请的空间 

例如：新建一个数组作为返回值 → `O(1)` 新建一个数组用于中间计算但不返回 → `O(n)`

**为什么重要**

面试和刷题都会要求分析复杂度，而且理解复杂度之后，你看到一道题会自然去想"有没有比 `O(n²)` 更好的解法"，这就是优化的驱动力。
