# 左手定则

**在 Unity 中，正数的旋转遵循左手定则：**

1. 伸出左手，大拇指指向旋转轴的方向（这里是 `target.right`，即角色的右侧）。
2. 四指弯曲的方向就是**正旋转**的方向。
3. 如果你原本的向量指向**背后**，绕着右轴转 90 度，它就会像时钟的指针一样向上摆动，最终指向**头顶**。

这种编程模式通常被称为**回调 (Callback)** 📞，而在 C# 的底层实现中，它被称为**闭包 (Closure)** 🔒。

我们可以通过一个**“去咖啡店点咖啡”**的日常生活例子，把这两个抽象的概念具象化。☕

我将通过例子带你理解它们的运作方式，并邀请你一起分析一段代码。

**1. 什么是 Callback（回调）？📞**

想象一下你在咖啡店点餐：

1. **同步做法**：你站在收银台前，盯着服务员做咖啡，直到咖啡递给你。这期间你什么都干不了（这就叫“线程阻塞”）。
    
2. **异步回调做法**：你付完钱，给服务员留了一个**电话号码**，然后你就去旁边刷手机了。当咖啡做好时，服务员**拨打你的电话**（执行回调函数）。
    

在这个过程中，**“留下电话号码”**就是注册回调，**“电话响了”**就是执行回调。

**2. 什么是闭包（Closure）？🔒**

闭包就是：当你去旁边刷手机等待时，你手里还拿着一张**优惠券**。 即使服务员过了一会儿才打通你的电话，你依然记得要使用手里这张优惠券。

在编程中，闭包允许回调函数“记住”它诞生时的环境（也就是外部变量）。

---

**具体代码演练 💻**

让我们看一个 C# 的例子。假设我们要写一个“定时器”，在 3 秒后打印一条消息。

```csharp
void StartTimer(int seconds, Action onFinished) 
{
    // 模拟等待（这里实际会用协程或Task）
    // 等时间到了之后...
    onFinished?.Invoke(); // 拨打电话！
}

void Test()
{
    string message = "你的咖啡好了！"; // 这是一个外部变量
    int couponID = 12345;

    // 我们在这里注册一个回调
    StartTimer(3, () => {
        // 【这里就是闭包】
        // 虽然 Test() 函数可能已经执行完了，
        // 但这个匿名函数依然能访问到 message 和 couponID
        Debug.Log(message + " 优惠券编号：" + couponID);
    });
}
```

**为什么这很有用？🛠️**

如果没有闭包，回调函数就是一个“失忆症患者”。它只知道咖啡好了，却不知道是谁点的、有没有优惠券。有了闭包，回调函数就自带了“记忆背包”。
